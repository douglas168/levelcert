# 圖表 3：決策樹分裂準則比較

> **用途：** Gini vs Entropy vs Information Gain 公式並排，標示計算成本差異。考試最常考「哪個計算速度較快」以及「sklearn 預設是哪個」。

---

## 三種分裂準則公式並排

```
┌─────────────────────────────────────────────────────────────────┐
│                    分裂準則（Split Criterion）                    │
├─────────────────┬────────────────────┬────────────────────────  │
│   Gini Impurity │     Entropy        │   Information Gain       │
│                 │                    │                          │
│ Gini(t) =       │ H(t) =             │ IG = H(parent)           │
│ 1 - Σ pᵢ²      │ -Σ pᵢ · log₂(pᵢ)  │   - Σ [nₖ/n · H(child)] │
│                 │                    │                          │
│ 範圍: [0, 0.5]  │ 範圍: [0, log₂(K)] │ 越大 → 分裂越有意義      │
│ （2-class 最大）│ K = 類別數          │                          │
├─────────────────┼────────────────────┼────────────────────────  │
│ ✅ 無 log 計算  │ ❌ 有 log 計算      │ 基於 Entropy 計算        │
│ ✅ 速度快       │ 速度慢              │ 速度最慢                  │
│ ✅ sklearn 預設 │ criterion='entropy' │ （同 Entropy 路徑）       │
└─────────────────┴────────────────────┴────────────────────────  ┘
```

---

## 數值範例：同一個節點，三種準則

**節點分佈：60 個 Class A，40 個 Class B（p_A=0.6, p_B=0.4）**

```
Gini = 1 - (0.6² + 0.4²)
     = 1 - (0.36 + 0.16)
     = 1 - 0.52
     = 0.48   ← 無 log，計算步驟少

Entropy = -(0.6 · log₂0.6 + 0.4 · log₂0.4)
        = -(0.6 · (-0.737) + 0.4 · (-1.322))
        = -(-0.442 - 0.529)
        = 0.971   ← 有 log，計算步驟多

Gini < Entropy  （數值不同，但選出的最佳分裂通常相同）
```

---

## 純節點 vs 最混雜節點比較

```
純節點（100% 同類）           最混雜節點（2-class 各50%）
pᵢ = [1.0, 0.0]              pᵢ = [0.5, 0.5]

Gini    = 1 - (1² + 0²) = 0  Gini    = 1 - (0.5² + 0.5²) = 0.5
Entropy = -(1·log₂1 + 0·log₂0) = 0  Entropy = -(0.5·log₂0.5 × 2) = 1.0
```

```
不純度
1.0 │         Entropy
0.5 │    ___/‾‾‾\___     Gini
0.0 │___/           \___
    └──────────────────── p (一類比例)
    0                  1
    ↑純            ↑純
```

Entropy 曲線比 Gini 更高 → 對不純度的懲罰更重 → 更敏感。

---

## Gini vs Entropy 決策表

| 比較維度 | Gini | Entropy |
|---|---|---|
| 公式 | 1 - Σpᵢ² | -Σpᵢ log₂pᵢ |
| 計算成本 | 低（無 log）✅ | 高（有 log）|
| 對不純度敏感度 | 較低 | 較高 |
| sklearn 預設 | ✅ `criterion='gini'` | `criterion='entropy'` |
| 結果差異 | 通常相同 | 偶爾選不同特徵 |
| 考試答案 | 「速度快」的答案 | 「更精確/敏感」的答案 |

---

## 考試快判

```
看到「計算速度較快」「計算成本較低」→ Gini
看到「sklearn 預設」→ Gini（criterion='gini'）
看到「對不純度更敏感」「更精確」→ Entropy
看到「Information Gain」→ 以 Entropy 為基礎計算
```

---

## 陷阱提醒

❌ **常見誤解：Entropy 和 Gini 建出的樹結果完全不同**
✅ **正解：** 絕大多數情況下兩者選出相同的最佳分裂，最終模型準確率差異極小。考試考的是「計算成本差異」而非「效果差異」。
