# 圖表 1：偏誤來源分類樹（Bias Source Taxonomy）

> 用途：一眼看清資料偏誤與模型偏誤的分類層次，以及每種偏誤發生在 ML pipeline 的哪個階段。

```
演算法偏誤（Algorithmic Bias）
│
├── 📊 資料偏誤來源（Data Bias Sources）
│   ├── 歷史偏誤（Historical Bias）
│   │   ├── 定義：訓練資料反映了歷史上的歧視或不平等
│   │   ├── 發生階段：資料收集前（世界本身就不公平）
│   │   └── 範例：男性主導的歷史履歷 → 模型偏好男性應聘者
│   │
│   ├── 代表性偏誤（Representation Bias）
│   │   ├── 定義：訓練集中某群體樣本數量不足
│   │   ├── 發生階段：資料收集 / 採樣
│   │   └── 範例：人臉辨識資料集 90% 為白人男性 → 深色膚色辨識率低
│   │
│   ├── 測量偏誤（Measurement Bias）
│   │   ├── 定義：特徵或標籤的測量方式對不同群體不一致
│   │   ├── 發生階段：資料標記 / 特徵工程
│   │   └── 範例：醫療紀錄中少數族群就診紀錄較少 → 健康分數低估
│   │
│   ├── 聚合偏誤（Aggregation Bias）        ← ⚠️ 邊界模糊
│   │   ├── 定義：單一模型套用異質性子群，忽略群體間差異
│   │   ├── 發生階段：資料準備決策 / 模型設計
│   │   ├── 學術歸類：Suresh & Guttag 2019 歸入模型/流程偏誤
│   │   └── 考試歸類：iPAS 情境多歸在資料準備決策偏誤
│   │
│   ├── 評估偏誤（Evaluation Bias）
│   │   ├── 定義：測試集或評估指標未代表目標群體
│   │   ├── 發生階段：模型評估
│   │   └── 範例：用整體 Accuracy 評估，忽略少數類別 F1
│   │
│   └── 部署偏誤（Deployment Bias）
│       ├── 定義：模型被用於設計範疇外的情境
│       ├── 發生階段：部署後
│       └── 範例：為犯罪率預測設計的模型，被用於決定釋放假設條件
│
└── 🤖 模型偏誤來源（Model Bias Sources）
    ├── 回饋迴圈（Feedback Loop）
    │   ├── 定義：模型輸出影響未來訓練資料，強化原有偏誤
    │   └── 範例：推薦系統 → 少數內容曝光更少 → 資料更少 → 更少推薦
    │
    ├── 代理變數（Proxy Variable）
    │   ├── 定義：用相關性特徵替代受保護屬性，間接引入偏誤
    │   └── 範例：郵遞區號 → 代理種族 → 貸款模型間接歧視
    │
    └── 群體評估差異（Evaluation Gap）
        ├── 定義：模型對不同群體的預測準確度差異顯著
        └── 範例：糖尿病預測模型對亞裔準確率比白人低 15%
```

## 考試速記

| 偏誤類型 | 關鍵詞 | 解決方向 |
|---|---|---|
| Historical | 歷史資料反映歧視 | 資料清洗 / 重新標記 |
| Representation | 樣本數量不足 | 過採樣 / 重採樣 |
| Measurement | 測量方式不一致 | 標記標準化 |
| Aggregation | 單一模型跨群體 | 分群模型 / 特徵工程 |
| Evaluation | 指標未覆蓋弱勢群體 | 分群評估 |
| Deployment | 超出設計範疇使用 | 使用政策限制 |
| Feedback Loop | 輸出影響未來輸入 | 主動採樣 / 多樣化探索 |
| Proxy Variable | 相關變數替代敏感屬性 | 移除代理特徵 |
