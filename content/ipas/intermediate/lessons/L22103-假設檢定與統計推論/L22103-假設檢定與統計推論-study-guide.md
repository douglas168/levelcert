## 1. Big Picture / Core Pipeline 🔥🔥🔥

### 先懂一句話

`假設檢定（Hypothesis Testing）` 是用樣本資料來判斷：「這個差異、效果或關聯，像不像只是自然波動？」

它不是證明真理，而是用一套固定規則做決策。

### Everyday Analogy

像老師懷疑某班這次平均分數真的變高了。老師不會只看幾個高分同學就說「一定進步」，而是先假設「其實沒有進步，只是抽到的樣本剛好比較高」，再看資料夠不夠強到推翻這個保守假設。

### 先問自己一個問題

```text
如果「沒有差異」是真的，現在看到的資料會不會太誇張？
```

如果太誇張，就傾向拒絕 `H0`。

如果不誇張，就只能說無法拒絕 `H0`。

### 技術說法

`假設檢定（Hypothesis Testing）` 從 `虛無假說（Null Hypothesis, H0）` 出發，再用樣本資料計算檢定統計量與 `p-value`。最後把 `p-value` 和 `顯著水準（Significance Level, α）` 比較，決定是否拒絕 `H0`。

### 流程

```text
商業 / 研究問題
→ 寫出 H0 / H1
→ 設定 α
→ 選擇 t / χ² / F 檢定
→ 算出檢定統計量與 p-value
→ 比較 p-value 與 α
→ 拒絕 H0 或無法拒絕 H0
```

### 比較表這樣讀

這張表不要死背名詞，要當作「看到題目關鍵字時的選擇表」。

| 題目看到 | 先想到 | 為什麼 |
| --- | --- | --- |
| 是否有差異、有效果、有關聯 | `H0 / H1` | 先把保守說法與替代說法寫清楚 |
| `p-value = 0.03`、`α = 0.05` | 比較 `p-value` 與 `α` | 決定是否拒絕 `H0` |
| `95% CI` 是否包含假設值 | CI 與檢定結論 | 雙尾檢定常可用 CI 快判 |
| 平均數比較 | `t-test` 或 `F-test / ANOVA` | 平均數是數值型資料 |
| 類別變數是否獨立 | `χ² test of independence` | 兩個變數都是類別型 |

本課邊界：

- 會考：假設檢定框架、`α / β / p-value / CI`、`t-test`、`χ² test`、`F-test / one-way ANOVA`。
- 不展開：非參數檢定、效果量、貝氏推論、迴歸、機器學習專屬統計、多因子 ANOVA 與事後檢定細部計算。

### 記憶方式

```text
假設檢定 = 先假設沒事，再看資料夠不夠反常
p-value = 在 H0 世界裡，這份資料有多罕見
α = 事前設定「多罕見才算罕見」的門檻
```

### Exam Rule

```text
p-value ≤ α → 拒絕 H0
p-value > α → 無法拒絕 H0
平均數比較 → t-test 或 F-test
類別 x 類別 / 是否獨立 → χ² test
三組以上平均數 → F-test / one-way ANOVA
```

### Quick Check

題目給 `α = 0.05`、`p-value = 0.08`，正確結論是什麼？

答案：無法拒絕 `H0`。因為 `0.08 > 0.05`，資料證據還不夠強。

## 2. 假設檢定框架：H0 / H1 / 單尾雙尾 🔥🔥🔥

### 先懂一句話

`H0` 是保守起點：沒有差異、沒有效果、沒有關聯。`H1` 是你想檢查的替代說法：有差異、有效果、有關聯。

### Everyday Analogy

像法庭中的「先假設無罪」。除非證據夠強，否則不會輕易推翻原本的保守立場。

### 先問自己一個問題

```text
題目原本要挑戰的保守說法是什麼？
```

通常那個保守說法就是 `H0`。

### 技術說法

`虛無假說（Null Hypothesis, H0）` 通常表示沒有差異、沒有效果、沒有關聯。`對立假說（Alternative Hypothesis, H1）` 表示研究者想找證據支持的方向，可能是不同、變大或變小。

### 流程 / 選擇流程

```text
讀題目
→ 找保守說法：沒有差異 / 無效果 / 獨立
→ 寫成 H0
→ 找題目想檢查的改變
→ 寫成 H1
→ 看 H1 有沒有方向，決定單尾或雙尾
```

### 比較表這樣讀

先看 `H0 / H1` 的角色，再看題目有沒有指定方向。

| 名稱 | 中文意思 | 題目常見句型 | 你要怎麼想 |
| --- | --- | --- | --- |
| `H0` 虛無假說（Null Hypothesis） | 無差異、無效果、無關聯 | 平均數相同、兩變數獨立 | 保守起點 |
| `H1` 對立假說（Alternative Hypothesis） | 有差異、有效果、有關聯 | 平均數不同、兩變數不獨立 | 想找證據支持的說法 |

| 情境 | `H0` | `H1` |
| --- | --- | --- |
| 新版派單流程是否改變平均送達時間 | 新舊平均送達時間相同 | 新舊平均送達時間不同 |
| 會員等級與回購是否有關 | 會員等級與回購彼此獨立 | 會員等級與回購不獨立 |

### 一步一步例子

題目：公司想知道新版訓練是否「提升」平均分數。

```text
Step 1：找保守說法
H0 = 新版訓練沒有提升平均分數

Step 2：找題目想證明什麼
H1 = 新版訓練提升平均分數

Step 3：判斷方向
「提升」代表大於
→ 右尾檢定
→ H1: μ > μ0
```

單尾與雙尾：

```text
雙尾檢定：H1: μ ≠ μ0   看是否不同，不管變大或變小
右尾檢定：H1: μ > μ0   看是否變大 / 提升
左尾檢定：H1: μ < μ0   看是否變小 / 降低
```

### 記憶方式

```text
H0 = 沒事
H1 = 有事
是否不同 = 雙尾
是否提升 / 降低 = 單尾
```

### Exam Rule

```text
無差異 / 無效果 / 無關聯 → H0
有差異 / 有效果 / 有關聯 → H1
是否不同 → 雙尾檢定
是否提升 / 大於 → 右尾檢定
是否降低 / 小於 → 左尾檢定
看到結果後才改尾巴 → 錯誤作法
```

### Quick Check

某公司想知道新版訓練是否「提升」平均分數，`H1` 應該偏向哪種形式？

答案：右尾檢定，`H1: μ > μ0`。因為題目指定方向是「提升」。

## 3. α、β、Type I / Type II Error 與 Power 🔥🔥🔥

### 先懂一句話

檢定一定可能犯錯。`α` 是「沒事卻誤判有事」，`β` 是「有事卻沒抓到」。

### Everyday Analogy

像防盜警報：

- `α`：其實沒有小偷，警報卻響了。這是 false positive。
- `β`：其實有小偷，警報卻沒響。這是 false negative。

### 先問自己一個問題

```text
我現在犯錯時，是把真的 H0 拒絕掉，還是把假的 H0 留下來？
```

### 技術說法

`第一類錯誤（Type I Error, α）` 是 `H0` 其實為真，但你拒絕 `H0`。`第二類錯誤（Type II Error, β）` 是 `H0` 其實為假，但你沒有拒絕 `H0`。`檢定力（Power）` 是 `1 - β`，表示真的有差異時成功抓到差異的能力。

### 流程

```text
先設定 α
→ 接受一定的 Type I error 風險
→ 做檢定決策
→ 同時理解 β 與 power 的取捨
```

### 比較表這樣讀

先看「真實世界」再看「你的決策」。考試常把這兩個方向混在一起問。

| 真實情況 / 你的決策 | 拒絕 `H0` | 無法拒絕 `H0` |
| --- | --- | --- |
| `H0` 真的為真 | Type I Error，`α` | 正確 |
| `H0` 其實為假 | 正確 | Type II Error，`β` |

| 名稱 | 白話 | 口訣 | 常見英文 |
| --- | --- | --- | --- |
| Type I Error, `α` | 沒有效果卻判成有效 | 誤拒真 | false positive |
| Type II Error, `β` | 有效果卻沒抓到 | 誤留假 | false negative |
| Power, `1 - β` | 真的有差時，成功抓到差異 | 抓真差 | statistical power |

### 一步一步例子

情境：`H0 = 新藥沒有效果`。

```text
Case A：新藥其實沒有效果，但檢定拒絕 H0
→ 你把「真的 H0」拒絕掉
→ Type I Error
→ α
→ false positive

Case B：新藥其實有效果，但檢定沒有拒絕 H0
→ 你把「假的 H0」留下來
→ Type II Error
→ β
→ false negative
```

`顯著水準（Significance Level, α）` 常見是 `0.05`。`α` 設得越小，越不容易亂拒絕 `H0`，但也可能更難抓到真的差異，讓 `β` 上升、`power` 下降。

### 記憶方式

```text
α = 誤拒真 = false positive
β = 誤留假 = false negative
Power = 1 - β = 抓到真差異的能力
```

### Exam Rule

```text
α → Type I Error → H0 真卻誤拒 → 誤拒真
β → Type II Error → H0 假卻沒拒絕 → 誤留假
Power → 1 - β → 真的有差時抓到差異的能力
false positive → 沒有效果卻判成有效
false negative → 有效果卻沒抓到
α 設很小 → 較不易誤拒 H0，但可能降低檢定力
```

### Quick Check

`H0` 其實是假的，但檢定結果沒有拒絕 `H0`，這是哪一種錯誤？

答案：Type II Error，`β`。因為這是「誤留假」。

## 4. p-value 與結論用語 🔥🔥🔥

### 先懂一句話

`p-value` 是在 `H0` 成立的世界裡，看到現在這種資料或更極端資料的機率。

```text
p-value 小 → 這份資料在 H0 的世界裡很罕見 → 傾向拒絕 H0
p-value 大 → 這份資料在 H0 的世界裡不奇怪 → 無法拒絕 H0
```

### Everyday Analogy

像全班平常平均 70 分，今天抽到一小組平均 95 分。`H0` 是「全班其實沒有變強，只是抽樣波動」。如果在這個 `H0` 世界裡，抽到 95 分或更高非常罕見，那 `p-value` 就會很小。

### 先問自己一個問題

```text
如果 H0 真的成立，現在這份資料有多罕見？
```

### 技術說法

`p-value` 不是 `H0` 為真的機率。它是假設 `H0` 成立後，觀察到目前結果或更極端結果的機率。決策時把它和事前設定的 `α` 比較。

### 流程

```text
算出 t / χ² / F 統計量
→ 轉成 p-value
→ 跟 α 比較
→ 拒絕 H0 或無法拒絕 H0
```

### 一步一步例子

```text
α = 0.05
p-value = 0.028

Step 1：先看門檻
α = 0.05
意思是：我事前設定 5% 作為「罕見到可以拒絕 H0」的門檻

Step 2：看資料罕見程度
p-value = 0.028
意思是：如果 H0 真的成立，看到目前結果或更極端結果的機率是 2.8%

Step 3：比較
0.028 ≤ 0.05

Step 4：下結論
拒絕 H0
→ 資料提供足夠證據，支持存在統計上顯著差異
```

正確結論句型：

- 可以說：在 `α = 0.05` 下，`p-value = 0.028`，小於 `0.05`，因此拒絕 `H0`。
- 可以說：資料在 `H0` 成立時很罕見，因此支持存在統計上顯著差異。
- 不要說：`H0` 被證明是假的。
- 不要說：`p-value = 0.028` 代表 `H0` 只有 2.8% 機率為真。

### 比較表這樣讀

這張表專門用來排除錯誤選項。

| 易混淆點 | 正確理解 |
| --- | --- |
| `p-value` 是 `H0` 為真的機率 | 錯。它是假設 `H0` 成立後，資料有多罕見 |
| `p-value` 很小代表效果很大 | 錯。它代表統計證據強，不直接代表效果大小 |
| `p-value > α` 代表證明 `H0` 為真 | 錯。只能說無法拒絕 `H0` |

### 記憶方式

```text
p-value = H0 世界裡的罕見程度
p ≤ α = 夠罕見，拒絕 H0
p > α = 不夠罕見，無法拒絕 H0
```

### Exam Rule

```text
p 小於或等於 α → reject H0
p 大於 α → fail to reject H0
題目出現「證明 H0 為真」→ 通常是陷阱
題目問 p-value 意義 → 一定要加上「在 H0 成立下」
```

### Quick Check

`p-value = 0.03` 是否表示 `H0` 為真的機率是 3%？

答案：不是。`p-value` 是在 `H0` 成立下，觀察到目前或更極端結果的機率。

## 5. 信賴區間 CI 與檢定關係 🔥🔥

### 先懂一句話

`信賴區間（Confidence Interval, CI）` 是用樣本資料估計母體參數可能落在哪個合理範圍；考試常看它有沒有包含 `H0` 的假設值。

### Everyday Analogy

像估算某職缺合理薪資區間。如果你估出合理薪資是 72K 到 79K，但公司宣稱基準是 70K，70K 不在合理範圍內，你就會懷疑原本基準。

### 先問自己一個問題

```text
H0 的假設值有沒有落在 CI 這個合理範圍裡？
```

### 技術說法

在雙尾檢定中，`95% 信賴區間（95% Confidence Interval）` 常可對照 `α = 0.05` 的檢定結論。若 `95% CI` 不包含 `H0` 假設值，通常對應拒絕 `H0`；若包含，通常對應無法拒絕 `H0`。

### 流程

```text
樣本資料
→ 估計參數範圍 CI
→ 找 H0 的假設值
→ 看假設值是否在 CI 內
→ 對照 p-value 結論
```

### 一步一步例子

```text
H0: μ = 70

Case A:
95% CI = [72.1, 79.4]

Step 1：找假設值
H0 假設值 = 70

Step 2：看 70 是否在 [72.1, 79.4] 內
70 < 72.1
→ 70 不在區間內

Step 3：結論
通常拒絕 H0
```

```text
Case B:
95% CI = [67.8, 73.5]

Step 1：找假設值
H0 假設值 = 70

Step 2：看 70 是否在 [67.8, 73.5] 內
67.8 ≤ 70 ≤ 73.5
→ 70 在區間內

Step 3：結論
通常無法拒絕 H0
```

> 注意：CI 與拒絕 `H0` 的快判關係，這裡指的是雙尾檢定（two-tailed test）。單尾檢定需使用對應的單尾 CI，不能直接拿雙尾 95% CI 硬判。

### 比較表這樣讀

這張表要當成「CI 快判表」。

| 題目給什麼 | 你要看什麼 | 快判 |
| --- | --- | --- |
| `95% CI` 與 `H0` 假設值 | 假設值是否在區間內 | 在裡面 → 通常無法拒絕 |
| `95% CI` 與 `H0` 假設值 | 假設值是否在區間內 | 不在裡面 → 通常拒絕 |
| CI 寬或窄 | 估計精確度 | 越窄通常越精確，但不是本課主要決策點 |

### 記憶方式

```text
CI = 合理範圍
H0 值在合理範圍內 → 無法拒絕
H0 值不在合理範圍內 → 拒絕
```

### Exam Rule

```text
95% CI 不含 H0 假設值 → 通常拒絕 H0
95% CI 含 H0 假設值 → 通常無法拒絕 H0
CI 是合理範圍，不是保證範圍
判斷時看「是否包含假設值」，不是只看區間寬窄
```

### Quick Check

`H0: μ = 100`，`95% CI = [102, 108]`，通常會拒絕還是無法拒絕 `H0`？

答案：通常拒絕 `H0`。因為假設值 `100` 不在信賴區間內。

## 6. t-test：平均數比較 🔥🔥🔥

### 先懂一句話

`t 檢定（t-test）` 是平均數比較工具：一組平均數、兩組平均數，或同一批人的前後平均差。

### Everyday Analogy

像比較班級成績：

- 一班平均是否等於 70 分？
- A 班和 B 班平均是否不同？
- 同一班上課前後是否進步？

這三種都是平均數問題，但資料關係不同，所以選的 t-test 不同。

### 先問自己一個問題

```text
這是在比較「幾組平均數」？資料是不同人，還是同一批人前後？
```

### 技術說法

`t-test` 用於平均數檢定。常見三種：`單一樣本 t 檢定（one-sample t-test）`、`獨立雙樣本 t 檢定（independent two-sample t-test）`、`配對 t 檢定（paired t-test）`。

### 選擇流程

```text
題目問平均數？
│
├─ 一組平均數 vs 標準值
│  └─ one-sample t-test
│
├─ 兩組平均數，不同人 / 不同群體
│  └─ independent two-sample t-test
│
└─ 同一批人前後 / 成對資料
   └─ paired t-test
```

### 比較表這樣讀

先看「資料關係」，不要只看有幾欄資料。

| t-test 類型 | 什麼時候用 | 典型例子 | 常見 SciPy |
| --- | --- | --- | --- |
| One-sample t-test | 一組平均數 vs 某標準值 | 平均上手天數是否等於 3 天 | `ttest_1samp` |
| Independent two-sample t-test | 兩組不同群體平均數 | A/B 廣告平均點擊數是否不同 | `ttest_ind` |
| Paired t-test | 同一批對象前後或成對資料 | 同一批員工導入工具前後處理時間 | `ttest_rel` |

### 一步一步例子

配對樣本的核心不是「兩欄資料」，而是「每一筆前後可以一一對應」。

```text
員工 A：導入前 18 分 → 導入後 14 分 → 差值 -4
員工 B：導入前 22 分 → 導入後 20 分 → 差值 -2
員工 C：導入前 16 分 → 導入後 12 分 → 差值 -4

Step 1：每個人都有前後兩筆
→ 是同一批人

Step 2：先算每個人的差值
-4, -2, -4

Step 3：paired t-test 檢查的是
平均差值是否為 0
```

讀碼提醒：

```python
from scipy.stats import ttest_ind, ttest_rel

stat, p = ttest_ind(group_a, group_b, equal_var=True)
stat, p = ttest_rel(before, after)
```

`ttest_ind` 的 `ind` 是 independent。若 `equal_var=False`，常稱為 Welch's t-test，表示不假設兩組變異數相等。

### 檢定前提假設

這張表是考試陷阱表：題目若問適用條件，要先檢查假設。

| 檢定 | 前提假設 |
| --- | --- |
| 獨立樣本 t 檢定 | 兩組獨立、各組近似常態（或 n ≥ 30）、等變異數；若不等可用 Welch's `equal_var=False` |
| 配對 t 檢定 | 差值近似常態分佈 |
| 單因子 ANOVA | 各組獨立、各組近似常態、各組變異數相等（homogeneity of variance） |

### 記憶方式

```text
one-sample = 一組 vs 標準值
independent = 不同人 vs 不同人
paired = 同一批人前後
```

### Exam Rule

```text
一組平均數 vs 標準值 → one-sample t-test
兩組不同人 / 不同群體平均數 → independent two-sample t-test / ttest_ind
同一批人前後 / paired data → paired t-test / ttest_rel
前後測、同用戶改版前後 → 不要選 ttest_ind
```

### Quick Check

同一批使用者在 App 改版前後各量一次操作時間，應優先想到哪種 t-test？

答案：`paired t-test`。因為資料是一一對應的同一批使用者前後比較。

## 7. χ² test 與 F-test / One-way ANOVA 🔥🔥🔥

### 先懂一句話

`χ² test` 看「類別和類別有沒有關」；`F-test / one-way ANOVA` 看「三組以上平均數是否整體有差」。

### Everyday Analogy

像選工具：

- 會員等級和是否回購有沒有關？這是類別 x 類別，用 `χ² test`。
- 北、中、南三區平均滿意度是否相同？這是三組平均數，用 `ANOVA`。

### 先問自己一個問題

```text
題目是在看類別關聯，還是在比較三組以上平均數？
```

### 技術說法

`卡方檢定（Chi-square Test, χ² test）` 在本課重點是 `獨立性檢定（Test of Independence）`。`F 檢定（F-test）` 在本課最重要的應用是 `單因子變異數分析（One-way ANOVA）`。

### 選擇流程

```text
題目不是 t-test
│
├─ 類別 x 類別 / 是否獨立 / 列聯表
│  └─ χ² test of independence
│
└─ 三組以上平均數
   └─ F-test / one-way ANOVA
```

### 比較表這樣讀

先看資料型態，再看題目問法。

| 題目看到 | 資料型態 | 選什麼 | 結論能說什麼 |
| --- | --- | --- | --- |
| 會員等級 vs 是否回購 | 類別 x 類別 | `χ² test of independence` | 兩變數是否獨立 / 有關聯 |
| 北中南三區平均滿意度 | 三組以上數值平均 | `F-test / one-way ANOVA` | 至少一組平均數是否不同 |
| ANOVA 顯著後問哪兩組不同 | 後續比較 | Post-hoc test | 需事後比較，本課不展開計算 |

### χ² test of independence

`χ² test` 用來判斷兩個類別變數是否有關聯。

```text
                有回購   無回購
白金會員          80       20
一般會員          45       55

H0：會員等級與是否回購獨立
H1：會員等級與是否回購不獨立
```

卡方檢定比較的是：

```text
Observed Count 實際觀察次數
vs
Expected Count 若真的獨立時理論上應該出現的次數
```

### χ² 一步一步例子

```text
Step 1：看資料型態
會員等級 = 類別
是否回購 = 類別

Step 2：看題目問什麼
問兩者是否有關 / 是否獨立

Step 3：選檢定
χ² test of independence

Step 4：看 p-value
p-value ≤ α → 拒絕 H0，表示兩者可能有關聯
p-value > α → 無法拒絕 H0，沒有足夠證據說兩者有關
```

讀碼提醒：

```python
from scipy.stats import chi2_contingency

stat, p, dof, expected = chi2_contingency(observed_table)
```

`observed_table` 應是觀察到的計數表（observed counts），不是直接丟百分比。

> 重要限制：當列聯表中任一格的期望次數（expected count）< 5 時，卡方檢定結果可能不可靠，應考慮 Fisher's Exact Test。

### F-test / One-way ANOVA

`F-test / one-way ANOVA` 用來比較三組以上平均數是否整體相同。

```text
H0: μ1 = μ2 = μ3 = ...
H1: 至少有一組平均數不同
```

`F` 的直觀：

```text
F = MS_between / MS_within
F = 組間均方 / 組內均方
```

### ANOVA 一步一步例子

情境：比較北、中、南三區平均滿意度是否相同。

```text
Step 1：看資料型態
滿意度 = 數值型
地區 = 三組：北、中、南

Step 2：看題目問什麼
問三組平均數是否相同

Step 3：寫假設
H0: μ北 = μ中 = μ南
H1: 至少有一區平均數不同

Step 4：選檢定
F-test / one-way ANOVA

Step 5：看 p-value
p-value ≤ α → 拒絕 H0，只能說至少一組不同
p-value > α → 無法拒絕 H0，沒有足夠證據說三區平均不同
```

| 情況 | 直觀結果 |
| --- | --- |
| 組平均很接近，組內波動又大 | `F` 小，`p` 大 |
| 組平均差很多，組內又穩定 | `F` 大，`p` 小 |

重要限制：如果 ANOVA 拒絕 `H0`，只能說「至少有一組平均數不同」，不能直接說是哪兩組不同。需使用事後比較（Post-hoc Test），例如 Tukey HSD，找出差異所在的組別對。考試題目需能識別此名詞，計算不在本課範圍內。

讀碼提醒：

```python
from scipy.stats import f_oneway

stat, p = f_oneway(group_a, group_b, group_c)
```

### 記憶方式

```text
χ² = 類別 x 類別
ANOVA = 三組以上平均數
ANOVA 顯著 = 至少一組不同，不是知道哪一組
```

### Exam Rule

```text
類別資料 / 列聯表 / 是否獨立 → χ² test of independence
χ² test 的輸入 → observed counts，不是百分比
三組以上平均數 → F-test / one-way ANOVA
ANOVA 拒絕 H0 → 至少一組不同，不代表知道哪一組不同
```

### Quick Check

題目問「會員等級」與「是否回購」是否有關，資料整理成列聯表，應選哪個檢定？

答案：`χ² test of independence`。因為兩個變數都是類別型，且題目問是否獨立 / 有關聯。

## 8. Exam Decision Trees 🔥🔥🔥

### 8.1 先決定要不要拒絕 H0

```text
題目給 p-value 與 α？
│
├─ p-value ≤ α
│  └─ 拒絕 H0
│
└─ p-value > α
   └─ 無法拒絕 H0
```

### 8.2 用 CI 快判

```text
題目給 95% CI 與 H0 假設值？
│
├─ 假設值在 CI 裡
│  └─ 通常無法拒絕 H0
│
└─ 假設值不在 CI 裡
   └─ 通常拒絕 H0
```

### 8.3 選檢定方法

```text
先問：題目要比較什麼？
│
├─ 平均數？
│  │
│  ├─ 一組平均數 vs 標準值
│  │  └─ one-sample t-test
│  │
│  ├─ 兩組平均數
│  │  │
│  │  ├─ 不同人 / 不同群體
│  │  │  └─ independent two-sample t-test
│  │  │
│  │  └─ 同一批人前後 / 成對資料
│  │     └─ paired t-test
│  │
│  └─ 三組以上平均數
│     └─ F-test / one-way ANOVA
│
└─ 類別 x 類別 / 是否獨立？
   └─ χ² test of independence
```

### 8.4 看題目用語選尾巴

```text
題目描述 H1 方向？
│
├─ 是否不同 / 是否改變
│  └─ 雙尾檢定
│
├─ 是否提升 / 是否大於
│  └─ 右尾檢定
│
└─ 是否降低 / 是否小於
   └─ 左尾檢定
```

## 9. Trap Clinic 🔥🔥🔥

### Trap 1：p-value 是 H0 為真的機率

錯。`p-value` 是在 `H0` 成立下，看到目前或更極端結果的機率。

Exam fix：

```text
題目問 p-value 意義 → 一定回答「在 H0 成立下」
```

### Trap 2：p > α 就證明 H0 為真

錯。`p > α` 只能說「無法拒絕 `H0`」，不是證明 `H0` 正確。

Exam fix：

```text
p > α → fail to reject H0
看到「證明 H0 為真」→ 多半是錯選項
```

### Trap 3：拒絕 H0 就代表 H1 被完全證明

錯。拒絕 `H0` 代表資料提供足夠證據支持 `H1`，但不是數學上的完全證明。

Exam fix：

```text
reject H0 → 有統計證據支持 H1
完全確定 / 證明 / 必然 → 注意陷阱
```

### Trap 4：α 和 β 方向背反

錯。`α` 是 `H0` 真卻誤拒；`β` 是 `H0` 假卻沒拒絕。

Exam fix：

```text
α → 誤拒真
β → 誤留假
power → 1 - β
```

### Trap 5：前後測資料用 ttest_ind

錯。`ttest_ind` 是兩組獨立樣本。前後測、同一批人、同用戶改版前後，應先想到 paired t-test。

Exam fix：

```text
same users before/after → paired t-test / ttest_rel
different groups → independent t-test / ttest_ind
```

### Trap 6：χ² test 可以直接吃百分比

錯。卡方獨立性檢定通常需要觀察到的計數表（observed counts），不是直接把百分比丟進去。

Exam fix：

```text
列聯表 + count → χ² test
只有百分比 → 小心，需回到計數資料
```

### Trap 7：ANOVA 顯著就知道哪一組不同

錯。One-way ANOVA 顯著只能說「至少有一組平均數不同」，不能直接判斷是哪兩組不同。

Exam fix：

```text
F-test / ANOVA reject H0 → 至少一組不同
哪組不同 → 需要後續比較，本課不展開
```

### Trap 8：看到結果再決定單尾或雙尾

錯。單尾或雙尾應根據研究問題事前決定，不能為了顯著才改方向。

Exam fix：

```text
是否不同 → 雙尾
是否提升 / 降低 → 單尾
看完結果才改尾巴 → 錯
```

## 10. Practice Questions

### 10.1 H0 / H1 與結論用語

**Q1.** 題目問新版流程是否「改變」平均送達時間，沒有指定變快或變慢，通常是單尾還是雙尾？

答案：雙尾。

理由：題目只問「是否改變 / 是否不同」，不指定方向。

**Q2.** `p-value = 0.04`、`α = 0.05`，應該如何決策？

答案：拒絕 `H0`。

理由：`0.04 ≤ 0.05`。

**Q3.** `p-value = 0.12`、`α = 0.05`，可以說「證明 H0 為真」嗎？

答案：不可以。

理由：只能說「無法拒絕 `H0`」，不是證明 `H0` 為真。

**Q4.** `p-value = 0.02` 正確代表什麼？

答案：在 `H0` 成立下，觀察到目前或更極端結果的機率是 0.02。

理由：`p-value` 不是 `H0` 為真的機率。

### 10.2 α、β、Power 與 CI

**Q5.** `H0` 其實為真，但研究者拒絕了 `H0`，這是什麼錯誤？

答案：Type I Error，`α`。

理由：`α = 誤拒真`。

**Q6.** `H0` 其實為假，但檢定沒有拒絕 `H0`，這是什麼錯誤？

答案：Type II Error，`β`。

理由：`β = 誤留假`。

**Q7.** `Power` 等於什麼？

答案：`1 - β`。

理由：Power 是真的有差異時，成功偵測到差異的能力。

**Q8.** `H0: μ = 50`，`95% CI = [52, 58]`，通常對應拒絕或無法拒絕 `H0`？

答案：拒絕 `H0`。

理由：假設值 `50` 不在區間內。

**Q9.** `H0: μ = 50`，`95% CI = [48, 55]`，通常對應拒絕或無法拒絕 `H0`？

答案：無法拒絕 `H0`。

理由：假設值 `50` 在區間內。

### 10.3 檢定方法選擇

**Q10.** 某平台宣稱平均等待時間是 5 分鐘，抽一批訂單檢查平均等待時間是否等於 5 分鐘，應選什麼？

答案：One-sample t-test。

理由：一組樣本平均數與一個標準值比較。

**Q11.** 比較 A 廣告組與 B 廣告組的平均轉單金額，兩組使用者互不重疊，應選什麼？

答案：Independent two-sample t-test。

理由：兩組獨立群體的平均數比較。

**Q12.** 同一批員工導入新系統前後各測一次平均處理時間，應選什麼？

答案：Paired t-test。

理由：同一批人前後資料是一一配對。

**Q13.** 比較北、中、南三區客服平均滿意度是否相同，應選什麼？

答案：F-test / One-way ANOVA。

理由：三組以上平均數的整體比較。

**Q14.** 檢查會員等級與是否回購是否獨立，應選什麼？

答案：Chi-square test of independence。

理由：兩個變數都是類別型，且題目問是否獨立 / 有關聯。

**Q15.** `chi2_contingency` 的輸入應該是什麼？

答案：觀察到的計數列聯表。

理由：卡方檢定比較 observed counts 與 expected counts，不是直接使用百分比。

### 10.4 Mixed Traps

**Q16.** One-way ANOVA 的 `p-value ≤ α`，可以直接說「第 1 組比第 3 組高」嗎？

答案：不可以。

理由：ANOVA 只能先說至少有一組平均數不同，不指出哪一組不同。

**Q17.** 題目出現 `scipy.stats.ttest_ind`，最可能對應哪種情境？

答案：兩組獨立樣本平均數比較。

理由：`ind` 代表 independent。

**Q18.** 題目出現 `scipy.stats.ttest_rel(before, after)`，最可能對應哪種情境？

答案：配對樣本 t-test。

理由：`rel` 表示 related，常見於同一批對象前後比較。

**Q19.** 題目只說「新方法是否讓平均分數不同」，可以因為樣本平均較高就改成右尾檢定嗎？

答案：不可以。

理由：單尾或雙尾要依研究問題事前決定，不能看結果後調整。

**Q20.** `α = 0.01` 比 `α = 0.05` 更嚴格，這一定比較好嗎？

答案：不一定。

理由：較小的 `α` 會降低誤拒 `H0` 的機率，但也可能讓真的差異更難被偵測到。

## Final Oral Recall

考前最後 3 分鐘，把這幾句唸一次：

1. 假設檢定先從 `H0` 開始，`H0` 通常代表無差異、無效果、無關聯。
2. `p-value ≤ α` 就拒絕 `H0`，`p-value > α` 就無法拒絕 `H0`。
3. `p-value` 不是 `H0` 為真的機率，而是在 `H0` 成立下看到目前或更極端結果的機率。
4. `α = 誤拒真`，`β = 誤留假`，`power = 1 - β`。
5. `95% CI` 不含假設值通常拒絕 `H0`，包含假設值通常無法拒絕 `H0`。
6. 平均想 `t`，類別想 `χ²`，三組以上平均想 `F / ANOVA`。
7. 前後測、同一批人、成對資料要想到 `paired t-test`，不要誤選 `ttest_ind`。
