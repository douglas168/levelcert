# GD vs SGD vs Mini-batch 收斂比較 (Convergence Comparison)

## 三種批次策略收斂路徑

```mermaid
graph LR
    subgraph GD["🔵 Full-batch GD<br/>批次大小 = 全部資料 N"]
        direction TB
        GD1["每次更新：計算所有 N 筆梯度"]
        GD2["梯度估計非常準確<br/>路徑：平滑直線"]
        GD3["問題：N 大時記憶體爆炸<br/>每步計算成本 O(N)"]
        GD1 --> GD2 --> GD3
    end

    subgraph SGD["🔴 SGD（隨機梯度下降）<br/>批次大小 = 1"]
        direction TB
        SGD1["每次更新：隨機抽 1 筆計算梯度"]
        SGD2["梯度估計雜訊大<br/>路徑：隨機震盪"]
        SGD3["優點：每步快，天然正則化<br/>缺點：難收斂到精確最小值"]
        SGD1 --> SGD2 --> SGD3
    end

    subgraph MB["🟢 Mini-batch SGD<br/>批次大小 = 32/64/128"]
        direction TB
        MB1["每次更新：隨機抽 B 筆計算梯度"]
        MB2["梯度估計適中<br/>路徑：可接受的震盪"]
        MB3["✅ 實際最常用<br/>GPU 並行友善，記憶體可控"]
        MB1 --> MB2 --> MB3
    end

    GD -.->|"雜訊 ↑，速度 ↑"| SGD
    GD -.->|"平衡點"| MB
```

## 三種策略比較表

| 特徵 | Full GD | SGD (B=1) | Mini-batch (B=32–256) |
|------|---------|-----------|----------------------|
| 批次大小 | 全部 N 筆 | 1 筆 | 32/64/128/256 筆 |
| 梯度估計準確度 | 🟢 最準確 | 🔴 雜訊最大 | 🟡 適中 |
| 每步計算成本 | 🔴 最高 O(N) | 🟢 最低 O(1) | 🟡 中 O(B) |
| 收斂穩定性 | 🟢 最穩定 | 🔴 震盪嚴重 | 🟡 可接受 |
| 記憶體需求 | 🔴 最大 | 🟢 最小 | 🟡 中（可調整） |
| GPU 並行效率 | 🟢 高（但受記憶體限制） | 🔴 幾乎無法並行 | 🟢 最佳 |
| 實際使用頻率 | 罕見（小資料集） | 少見 | ✅ 最常用 |

> 🔑 考試快判：「記憶體不足（OOM）」→ 減小 batch_size；「GPU 利用率低」→ 增大 batch_size（在記憶體限制內）
