# 降維方法選擇決策樹 — Dimensionality Reduction Decision Tree

## 決策樹

```
需要降維？
     │
     ▼
目的是什麼？
     │
     ├──► 預處理／加速後續模型
     │         │
     │         ▼
     │    資料是線性結構嗎？
     │         │
     │         ├──► 是 → PCA ✅
     │         │         (快速、可逆、保留全域結構)
     │         │
     │         └──► 否 → Kernel PCA
     │                   (非線性 kernel 版 PCA)
     │
     ├──► 視覺化（2D / 3D 散點圖）
     │         │
     │         ▼
     │    需要對新樣本單獨 transform？
     │         │
     │         ├──► 是 → UMAP ✅
     │         │         (有 transform()，速度快於 t-SNE)
     │         │
     │         └──► 否 → t-SNE ✅
     │                   (無 transform()，視覺效果佳)
     │
     └──► 推薦系統 / 文字語意（矩陣分解）
               │
               ▼
           SVD / 低秩近似 ✅
           (任意 m×n 矩陣，LSA, 協同過濾)
```

## 方法比較總表

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  特性     │   PCA    │   SVD    │  t-SNE   │  UMAP    │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│ 線性/非線性│   線性   │   線性   │  非線性  │  非線性  │
│ 可逆      │    ✅    │    ✅    │    ❌    │    ❌    │
│ transform()│   ✅    │    ✅    │    ❌    │    ✅    │
│           │          │          │ (無此方法)│(有此方法)│
│ 速度      │    快    │    快    │    慢    │    中    │
│ 保留結構  │  全域線性 │  全域線性 │  局部鄰域 │  局部+全域│
│ 監督/無監督│  無監督  │  無監督  │  無監督  │  無監督  │
│ 適用目的  │ 預處理   │矩陣分解  │  視覺化  │視覺化+下游│
│ 輸入矩陣  │  方陣    │ 任意m×n  │  任意    │  任意    │
│           │(共變異數)│          │          │          │
└──────────┴──────────┴──────────┴──────────┴──────────┘

⚠️  重要區別：
    t-SNE → 沒有 transform()，不能對新樣本單獨轉換
    UMAP  → 有 transform()，可以對新樣本單獨轉換
```

## 考試場景快判

```
場景 → 選擇
─────────────────────────────────────────────────────
「將 10 維客戶資料壓縮為 2 維以加速 K-Means」→ PCA
「Netflix 電影推薦：用戶-電影矩陣分解」→ SVD (低秩近似)
「把 MNIST 手寫字 784 維視覺化為 2D 散點圖」→ t-SNE 或 UMAP
「先視覺化、再對新上傳圖片單獨轉換」→ UMAP (有 transform())
「無監督，需要之後 pca.inverse_transform() 重建」→ PCA
「任意 m×n 矩陣做壓縮，不要求矩陣是方陣」→ SVD
```
