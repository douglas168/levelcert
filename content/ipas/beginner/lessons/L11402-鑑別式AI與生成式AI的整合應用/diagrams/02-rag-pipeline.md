# 圖解 02：RAG 流程（Retrieval-Augmented Generation Pipeline）

> 對應 Study Guide Section 3.5

---

## 完整 RAG 流程圖

```
  ┌────────────────────────────────────────────────────────────┐
  │                    使用者問題輸入                            │
  │         「我們公司的差旅報帳上限是多少？」                     │
  └───────────────────────────┬────────────────────────────────┘
                               ↓
  ═══════════════════════════════════════════════════════════════
   STEP 1：Retrieval（R）— 比對/篩選   ← 鑑別式任務
  ═══════════════════════════════════════════════════════════════
  ┌────────────────────────────────────────────────────────────┐
  │  知識庫（企業文件庫）                                        │
  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
  │  │ 人事規章  │ │ 差旅政策  │ │ 財務手冊  │ │ ...文件   │      │
  │  └──────────┘ └──────────┘ └──────────┘ └──────────┘      │
  │                    ↑ 搜尋比對                               │
  │           找出最相關的文件段落                               │
  └────────────────────────┬───────────────────────────────────┘
                            ↓ 相關文件段落（如：差旅政策第3條...）
  ═══════════════════════════════════════════════════════════════
   STEP 2：Augmentation（A）— 上下文增強
  ═══════════════════════════════════════════════════════════════
  ┌────────────────────────────────────────────────────────────┐
  │  組合提示詞（Prompt Assembly）                              │
  │                                                            │
  │  [系統指令] 你是企業HR助理，根據以下文件回答問題。           │
  │  [文件內容] 差旅報帳規定：國內出差每日上限NT$3,000...        │  ← 注入檢索內容
  │  [使用者問題] 差旅報帳上限是多少？                           │
  └────────────────────────┬───────────────────────────────────┘
                            ↓ 完整提示詞
  ═══════════════════════════════════════════════════════════════
   STEP 3：Generation（G）— 生成回答   ← 生成式任務
  ═══════════════════════════════════════════════════════════════
  ┌────────────────────────────────────────────────────────────┐
  │  大型語言模型（LLM）                                         │
  │  根據注入的文件內容，生成自然語言回答                          │
  └────────────────────────┬───────────────────────────────────┘
                            ↓
  ┌────────────────────────────────────────────────────────────┐
  │  最終回答（有文件依據，可溯源）                              │
  │  「根據公司差旅政策，國內出差每日報帳上限為NT$3,000，        │
  │   國外出差另有不同標準，請參閱差旅政策第3條。」               │
  └────────────────────────────────────────────────────────────┘
```

---

## RAG vs 純生成式 AI 對比

```
  同一問題：「差旅報帳上限是多少？」
                    ↓
  ┌─────────────────────────────────────────────────┐
  │  純生成式AI（無知識庫）                           │
  │  依賴訓練資料中的記憶回答                          │
  │  → 可能回答：「一般企業標準約NT$2,000-5,000」      │  ← 幻覺風險！可能不正確
  │    或完全捏造一個看起來合理但不存在的數字            │
  └─────────────────────────────────────────────────┘

                    vs

  ┌─────────────────────────────────────────────────┐
  │  RAG 系統（有知識庫）                             │
  │  先查公司最新文件，再根據文件回答                   │
  │  → 回答：「根據差旅政策第3條：NT$3,000（附來源）」  │  ← 準確、可溯源
  └─────────────────────────────────────────────────┘
```

---

## RAG 的優點與限制（考試重點）

| 面向 | 優點 | 限制 |
|------|------|------|
| 幻覺控制 | 降低（有文件依據） | 無法完全消除（仍可能誤讀文件） |
| 知識更新 | 更新知識庫即可，無需重訓模型 | 需維護知識庫（成本） |
| 可溯源性 | 可設計為附來源引用 | 依賴知識庫品質 |
| 延遲 | — | 多了一個 Retrieval 步驟（增加延遲） |
| 適用場景 | 需要即時/最新/精確資訊 | 不適合需要模型內化知識的任務 |

---

## 考試記憶卡

```
  RAG = R + A + G

  R = Retrieval（檢索/比對）
      ↳ 鑑別式任務：在知識庫中比對篩選相關資料

  A = Augmented（增強）
      ↳ 把找到的資料注入提示詞上下文

  G = Generation（生成）
      ↳ 生成式任務：LLM 根據資料生成自然語言回答

  口訣：「查（R）→ 塞（A）→ 答（G）」
  記住：「查是鑑別式任務，答是生成式」
```

> 注意：底層向量搜尋、embedding 技術屬中級範圍，初級只需理解「搜尋比對」的概念
