# 圖解 01：五種整合模式（Integration Patterns）

> 對應 Study Guide Section 2 / Section 3.2–3.7

---

## 1. 串聯模式（Pipeline Integration）

模組依序串接，前一個輸出是後一個輸入。可以是鑑別→生成，也可以是生成→鑑別。

**模式 A：鑑別→生成（先偵測再生成）**

```
 輸入
  ↓
┌─────────┐
│ 鑑別式AI │  偵測/分類/辨識
└────┬────┘
     ↓ 結果（標籤/分數/偵測結果）
┌─────────┐
│ 生成式AI │  根據鑑別結果生成內容
└────┬────┘
     ↓
 輸出（報告/回覆/摘要）
```

例子：腫瘤偵測（鑑別）→ 診斷報告生成（生成）

**模式 B：生成→鑑別（先生成後審核）**

```
 輸入
  ↓
┌─────────┐
│ 生成式AI │  創造/撰寫/生成內容
└────┬────┘
     ↓ 草稿
┌─────────┐
│ 鑑別式AI │  審核/評分/合規檢查
└────┬────┘
 合格 ↓    ↑ 不合格 → 退回重生成
 輸出（發布）
```

例子：文案生成（生成）→ 合規審核（鑑別）→ 不合格退回

**最經典案例：ASR→NLP→TTS 客服管線**

```
 客戶語音輸入
      ↓
┌────────────┐   聲音→文字
│ ASR 語音辨識│   鑑別式
└─────┬──────┘
      ↓ 文字
┌────────────┐   意圖分類（鑑別）+ 回覆生成（生成）
│  NLP 處理  │
└─────┬──────┘
      ↓ 回覆文字
┌────────────┐   文字→聲音
│ TTS 語音合成│   生成式
└─────┬──────┘
      ↓
 語音回覆輸出
```

## 2. 並聯模式（Parallel Integration）

兩條分支同時對同一輸入執行，互不依賴，最後融合輸出。

```
              輸入（同一份資料）
              ↙             ↘
┌─────────────────┐   ┌─────────────────┐
│   分支 A        │   │   分支 B        │
│   鑑別式AI      │   │   生成式AI      │
│  分類/評分/偵測 │   │  摘要/生成/描述 │
└────────┬────────┘   └────────┬────────┘
         └─────────┬───────────┘
                   ↓
           融合輸出（標籤 + 摘要）
```

例子：客戶評論同時進行情感分析（鑑別）+ 重點摘要生成（生成）

## 3. 強化模式（Augmentation）

生成式AI先合成訓練資料，幫助鑑別式AI學習稀有類別。

```
  問題：鑑別式AI缺乏罕見類別訓練資料
           ↓
  ┌──────────────────────────────┐
  │ 生成式AI（如 GAN）            │
  │ 合成大量稀有類別假資料         │
  └─────────────┬────────────────┘
                ↓ 合成資料
  ┌─────────────────────────────┐
  │  真實資料 ＋ 合成資料 混合    │
  │          訓練集              │
  └─────────────┬───────────────┘
                ↓
  ┌─────────────────────────────┐
  │ 鑑別式AI 訓練                │
  │ 效果大幅提升（尤其稀有類別）  │
  └─────────────────────────────┘
```

例子：GAN 合成工廠瑕疵圖片 → 混入真實資料 → 訓練品檢鑑別模型

## 4. RAG 模式（Retrieval-Augmented Generation）

先在知識庫搜尋比對相關文件（鑑別式任務），再根據文件生成回答。

```
  使用者提問
       ↓
  ┌─────────────────────────────┐
  │ R：Retrieval（比對/篩選）    │  ← 鑑別式任務
  │ 搜尋知識庫，找出相關文件段落  │
  └──────────────┬──────────────┘
                 ↓ 相關文件段落
  ┌─────────────────────────────┐
  │ A：Augmented（增強）        │
  │ 把文件內容注入提示詞（上下文）│
  └──────────────┬──────────────┘
                 ↓ 帶有文件上下文的提示詞
  ┌─────────────────────────────┐
  │ G：Generation（生成）       │  ← 生成式任務
  │ LLM 根據文件生成自然語言回答 │
  └──────────────┬──────────────┘
                 ↓
           最終回答（有文件依據）
```

## 5. 生—鑑—監 閉環（Generate-Discriminate-Monitor Loop）

生成→鑑別審核→監控效果→回饋改善，形成持續優化循環。

```
  ┌──────────────────────────────────┐
  │  1. Generate（生成）              │
  │     生成式AI 產出內容              │
  └──────────────┬───────────────────┘
                 ↓ 草稿/初稿
  ┌──────────────────────────────────┐
  │  2. Discriminate（鑑別審核）      │
  │     鑑別式AI 評估品質/合規性       │
  └──────┬───────────────────────────┘
 不合格  ↓ 合格
  退回重生成 → 發布/輸出
         ↓
  ┌──────────────────────────────────┐
  │  3. Monitor（監控）               │
  │     收集效果數據，異常回饋給生成端  │
  └──────────────┬───────────────────┘
                 ↓ 改善訊號
  回到步驟 1（持續迭代）
```

例子：行銷文案生成 → AI 合規審核 → 上線後監控點擊率 → 數據反饋改善生成策略

---

> 口訣：**串（依序）→ 並（同時）→ 強（先生成訓練資料）→ RAG（先查再答）→ 生鑑監（生成審核閉環）**
