# 資料來源地圖（Data Sources Map）

```
資料來源（Data Sources）
│
├── 內部資料（Internal Data）
│   └── 企業或組織「自己產生、自己擁有」的資料
│   │
│   ├── 企業資料庫（Database）
│   │   └── 訂單紀錄、庫存、客戶資料
│   │   └── 例：7-11 POS 系統每日銷售紀錄
│   │
│   ├── CRM 系統（Customer Relationship Management，客戶關係管理）
│   │   └── 客戶聯絡歷史、購買偏好
│   │   └── 例：保險公司的客戶保單資料
│   │
│   ├── ERP 系統（Enterprise Resource Planning，企業資源規劃）
│   │   └── 財務、供應鏈、人力資源資料
│   │   └── 例：製造業的庫存和生產排程
│   │
│   ├── App 日誌（Application Log）
│   │   └── 應用程式自動記錄的操作紀錄
│   │   └── 例：Uber Eats 記錄每筆訂單的下單時間和配送路線
│   │
│   └── IoT 感測器（IoT, Internet of Things，物聯網 感測器）
│       └── 物聯網裝置蒐集的即時數據
│       └── 例：工廠溫濕度感測器、智慧手環心率數據
│
└── 外部資料（External Data）
    └── 從組織「外部取得」的資料
    │
    ├── 🔥 政府開放資料（Open Government Data）
    │   └── 政府依開放資料精神公開的免費資料集
    │   └── 台灣：data.gov.tw（提供 CSV / JSON / XML）
    │   └── 九大優先領域：農業、空間、氣候、災防、交通、醫療、能源、社會、企業永續
    │
    ├── API 服務（Application Programming Interface，應用程式介面）
    │   └── 透過程式化介面取得第三方即時資料
    │   └── 例：中央氣象署 API、Google Maps API
    │
    ├── 網路公開資料（Web Data）
    │   └── 從公開網頁取得的資訊
    │   └── 例：新聞網站、維基百科、公開論壇
    │
    ├── 學術資料集（Academic Datasets）
    │   └── 學術機構公開的研究用標準資料集
    │   └── 例：Kaggle、UCI Machine Learning Repository
    │
    └── 商業資料供應商（Commercial Data Providers）
        └── 付費購買的市場研究資料
        └── 例：市調公司的消費者行為報告
```

## 內部 vs 外部 快速對照

| 比較項目 | 內部資料（Internal） | 外部資料（External） |
|---------|--------------------|--------------------|
| 來源 | 組織自己系統產生 | 從外部取得 |
| 擁有權 | 組織自己擁有 | 第三方擁有或公開 |
| 品質控制 | 較容易掌握 | 品質參差不齊 |
| 成本 | 系統建置和維護成本 | 可能免費（開放資料）或付費 |
| 代表例 | CRM、ERP、App Log、IoT | data.gov.tw、API、Kaggle |

## 🔑 考場快判：看到關鍵字 → 判斷來源

| 題目關鍵字 | 資料來源 |
|-----------|---------|
| CRM、ERP、公司資料庫 | 內部 |
| App Log、操作紀錄、使用者行為 | 內部 |
| IoT（物聯網）感測器、智慧裝置 | 內部 |
| data.gov.tw、政府開放資料 | 外部 |
| API（氣象、地圖、金融行情） | 外部 |
| Kaggle、學術資料集、UCI | 外部 |
| 市調公司報告、商業資料 | 外部 |

> 🔥 **考試重點：** 即使是「免費」的資料（如 data.gov.tw），對公司而言仍是「外部資料」——判斷的關鍵是「資料是否由公司自己的系統產生」，而非「是否付費」。
