# Diagram 03 — 公民開發者角色與影子 IT 風險

```
組織人員光譜（技術能力由低到高）
───────────────────────────────────────────────────────────────

  一般業務用戶          公民開發者                IT 開發者
  (Business User)    (Citizen Developer)       (IT Developer)
  ┌───────────────┐  ┌─────────────────────┐  ┌────────────────┐
  │               │  │                     │  │                │
  │  使用現有工具   │  │  使用 NC/LC 工具     │  │  傳統程式開發   │
  │  不做開發      │  │  自行建構業務應用     │  │  核心系統架構   │
  │               │  │                     │  │                │
  │ Excel / Word  │  │ Power Apps / Zapier  │  │ React / Java   │
  │ LINE / Email  │  │ Airtable / AppSheet  │  │ Python / SQL   │
  │               │  │                     │  │                │
  └───────────────┘  └─────────────────────┘  └────────────────┘
                              │
                              │ Gartner 定義公民開發者的兩個條件
                              │
                    ┌─────────┴───────────────┐
                    │                         │
              ✅ IT 部門以外                ✅ 使用組織核可工具
              （業務單位，非 IT 科）         （Sanctioned Tools）
                    │                         │
                    └─────────┬───────────────┘
                              │
                              ▼
                    ┌─────────────────────┐
                    │   合規公民開發者      │
                    │  ─────────────────  │
                    │  - 在 IT 治理框架內  │
                    │  - 使用核可平台      │
                    │  - IT 知情並管控     │
                    └─────────────────────┘

───────────────────────────────────────────────────────────────

⚠️  影子 IT（Shadow IT）風險示意

  觸發條件：業務人員「自行」下載未核可工具 / 繞過 IT 審核

  ┌──────────────┐  繞過 IT ❌  ┌──────────────────────┐
  │  業務人員     │ ──────────→ │  未核可工具/雲端服務   │
  │  （無技術背景）│             │  存放公司資料         │
  └──────────────┘             └──────────────────────┘
                                          │
                              ┌───────────┴───────────┐
                              │       風險清單         │
                              ├───────────────────────┤
                              │ ❌ 資料外洩（未加密）   │
                              │ ❌ 合規違反（GDPR 等）  │
                              │ ❌ IT 無法稽核管控     │
                              │ ❌ 資料孤島難整合      │
                              └───────────────────────┘

───────────────────────────────────────────────────────────────

公民開發者 vs 影子 IT 判斷口訣：

  核可 ✅ + IT 知情 ✅  →  公民開發者（合規）
  未核可 ❌ + IT 不知情 ❌ →  影子 IT（風險！）
```

> 考試關鍵：「IT 部門外」不等於影子 IT。
> 公民開發者雖然在 IT 部門外，但使用的是組織**核可**的工具，IT 知情且在治理框架內。
> 影子 IT 才是問題——員工自行繞過 IT 審核部署工具。
