# Diagram 01 — NC / LC / 傳統開發 三向比較

```
┌────────────────────────────────────────────────────────────────────────────────┐
│              No Code / Low Code / 傳統開發 — 五維比較                           │
│                                                                                │
│  維度         No Code（NC）       Low Code（LC）      傳統開發                  │
│  ─────────────────────────────────────────────────────────────────────────    │
│                                                                                │
│  技術門檻      ⬛⬜⬜⬜⬜ 低        ⬛⬛⬛⬜⬜ 中         ⬛⬛⬛⬛⬛ 高              │
│              （零程式能力）        （基礎技術知識）      （專業工程師）              │
│                                                                                │
│  開發速度      ⬛⬛⬛⬛⬛ 最快       ⬛⬛⬛⬛⬜ 快          ⬛⬛⬜⬜⬜ 最慢           │
│              （分鐘～天）          （天～週）            （週～月）                │
│                                                                                │
│  彈性／客製化   ⬛⬜⬜⬜⬜ 最低      ⬛⬛⬛⬜⬜ 中          ⬛⬛⬛⬛⬛ 最高           │
│              （受限平台模板）       （可加程式碼擴展）    （完全自由）               │
│                                                                                │
│  維護成本      ⬛⬜⬜⬜⬜ 最低      ⬛⬛⬜⬜⬜ 低至中       ⬛⬛⬛⬛⬛ 最高           │
│              （平台負責底層）       （平台 + 部分自管）   （團隊全自行維護）          │
│                                                                                │
│  適用場景      原型驗證            中型企業應用          核心系統                 │
│              流程自動化            需 API 整合的場景      高安全需求               │
│              非核心業務 App        公民開發者主導         高複雜邏輯               │
│                                                                                │
│  代表平台      Bubble              Power Apps            React + Node.js         │
│              Airtable             OutSystems            Python / Java           │
│              Zapier               Mendix                Spring Boot             │
│              AppSheet             Power Automate        自建後端 API             │
│                                                                                │
│  目標用戶      非技術業務人員        開發者                專業軟體工程師            │
│              （行銷、業務、教師）    公民開發者              資深架構師               │
│                                                                                │
└────────────────────────────────────────────────────────────────────────────────┘

速記口訣：
  速度  NC > LC > 傳統     （越早越快）
  彈性  傳統 > LC > NC     （越晚越彈）
  → 速度排序和彈性排序剛好相反！
```

> 考試重點：NC/LC 比傳統開發快且門檻低，但彈性不如傳統開發。
> LC 是 NC 和傳統開發的「中間地帶」——比 NC 彈性，比傳統快。
