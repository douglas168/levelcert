# Diagram 02 · Prompting Techniques Spectrum

## 複雜度光譜

```
  ← 簡單任務                                         複雜任務 →
  ← 低 token 消耗                                  高 token 消耗 →
  ← 無需範例                                          需要引導 →

  Zero-Shot      Few-Shot         CoT              ToT
  零樣本提示      少樣本提示       思維鏈提示        思維樹提示

  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐
  │不給任何   │  │給 2-5    │  │引導模型   │  │同時探索多條  │
  │範例，直   │  │個範例，  │  │展示逐步  │  │推理路徑，選  │
  │接下指令  │  │引導格式  │  │推理過程  │  │出最佳方案   │
  └──────────┘  └──────────┘  └──────────┘  └──────────────┘
       │              │             │               │
  ✅ 適合          ✅ 適合       ✅ 適合          ✅ 適合
  翻譯/摘要        格式統一      數學推理         多方案比較
  簡單問答         情緒分類      邏輯分析         策略規劃
                  批改報告      複雜計算         路線比較
```

## CoT vs ToT 關鍵差異

```
CoT（思維鏈）— 單條推理路線
─────────────────────────────────
問題
  │
  ▼
步驟 1
  │
  ▼
步驟 2
  │
  ▼
步驟 3
  │
  ▼
答案                   ← 一條路走到底


ToT（思維樹）— 多條推理路線
─────────────────────────────────
問題
  │
  ├──── 路徑 A ──── 結論 A
  │        評分: 60分
  │
  ├──── 路徑 B ──── 結論 B  ✅ 最佳
  │        評分: 90分
  │
  └──── 路徑 C ──── 結論 C
           評分: 75分
                             ← 比較後選最佳路徑
```

## 組合使用示例

```
Zero-Shot CoT:
  「請回答以下問題，並讓我們一步一步想：
   5 隻雞 5 天生 5 顆蛋，100 隻雞 100 天生幾顆蛋？」
   → 不需要範例，只需加上引導語

Few-Shot CoT:
  「例：Q: 2+3×4?  A: 先算乘法 3×4=12，再加法 2+12=14
   現在：Q: 5+6×7?  A: ???」
   → 同時給範例 + 要求推理步驟
```

## 陷阱速查

| 陷阱 | 正解 |
|------|------|
| CoT = ToT（都是推理）| CoT 是**單一推理鏈**，ToT 是**多分支推理樹** |
| Few-Shot = Fine-tuning | Few-Shot 在推論時提供範例，**不修改模型** |
| Zero-Shot CoT 需要給範例 | Zero-Shot CoT **不需要**範例，只需加引導語 |
| ToT 是 CoT 的簡化版 | ToT 是 CoT 的**延伸**（更複雜，不是簡化）|
