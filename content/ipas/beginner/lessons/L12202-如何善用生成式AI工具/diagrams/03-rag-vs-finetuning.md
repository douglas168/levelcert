# Diagram 03 · RAG vs Fine-tuning

## 核心比喻

```
  RAG（檢索增強生成）                Fine-tuning（微調）
  ─────────────────                  ──────────────────
  考試時可以翻書的                    考試前針對特定風格
     「開卷考試」                        做過特訓的
                                       「閉卷考試」

  查外部知識庫                        知識/風格內化
  不改模型參數                        修改模型參數
```

## 場景判斷速查

```
場景描述                    → 選擇

客服機器人查詢每週更新的        → RAG
產品規格手冊

讓 AI 學會用公司特定的          → Fine-tuning
分析師語氣和行為模式

讓 AI 回答基於最新法規的        → RAG
問題（法規每月更新）

讓 AI 對外輸出符合品牌          → Fine-tuning
固定語調的文案

知識常變動、需引用來源           → RAG

知識/風格需要穩定、內化          → Fine-tuning
到模型行為中
```

## 陷阱速查

| 陷阱 | 正解 |
|------|------|
| RAG 需要訓練模型 | RAG **不修改**模型參數 |
| Fine-tuning 適合動態資料 | Fine-tuning 更新成本高，**不適合**頻繁更新 |
| RAG 能完全消除幻覺 | RAG 只能**降低**幻覺，無法完全消除 |
| Fine-tuning = 重新訓練 | Fine-tuning 是在現有模型基礎上微調參數，不是從頭訓練 |
