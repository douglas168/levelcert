## Section 3 · 核心概念

### 3.1 線性迴歸（Linear Regression）

**先懂一句話：** 找一條「最接近所有資料點」的直線，用來預測連續數值。

**概念說明：**
線性迴歸假設輸入特徵和輸出值之間有「近似線性」關係。訓練的本質是找一條直線（或高維超平面），使得這條線與所有訓練資料點的「距離誤差總和」最小——這個方法叫做**最小平方法（Ordinary Least Squares, OLS）**，初級只需知道名字，不需計算。

| 面向 | 說明 |
|---|---|
| 適用任務 | **迴歸**（預測連續數值） |
| 輸出型態 | 連續數值（如房價 NT$1,200 萬） |
| 優點 | 簡單易解釋、速度快、適合線性關係資料 |
| 缺點 | 只能捕捉線性關係；對離群值（outlier）敏感 |
| 資料需求 | 小到中型資料即可 |
| 解釋性 | ⭐⭐⭐⭐⭐（最高） |

**典型用途（Taiwan 情境）：**
- 根據坪數、地段、屋齡預測房價
- 根據廣告投入預測銷售額
- 根據歷史用電量預測下個月的電費

🗣️ **白話說明：** 想像你記錄了 100 棟房子的「坪數」和「成交價格」，畫在 XY 座標上。線性迴歸就是找一條最能「穿過這些點中間」的直線，然後用這條線來預測新房子「大概值多少錢」。

**Quick check：** 線性迴歸適合「預測類別」還是「預測數值」？ → 預測數值（連續值），用在迴歸任務。

### 3.2 邏輯迴歸（Logistic Regression）

**先懂一句話：** 名字有「迴歸」，但其實是**分類**演算法——預測「屬於某個類別的機率」。

**概念說明：**
邏輯迴歸輸出的不是連續數值，而是「某件事發生的機率（0 到 1 之間）」，通常用在**二元分類**（是/否）問題。它用了一個叫做 **Sigmoid 函數**的概念，把任意數值壓縮到 0–1 之間，讓輸出可以解讀為機率。初級不需要知道 Sigmoid 公式，只需知道它的作用。

| 面向 | 說明 |
|---|---|
| 適用任務 | **分類**（尤其是二元分類）|
| 輸出型態 | 機率值 → 轉換成類別（0.5 為閾值） |
| 優點 | 簡單、快速、輸出有機率意義、可解釋 |
| 缺點 | 只能捕捉線性邊界，對非線性問題表現差 |
| 資料需求 | 小到中型均可 |
| 解釋性 | ⭐⭐⭐⭐⭐（最高） |

**典型用途（Taiwan 情境）：**
- 垃圾郵件過濾（是/不是垃圾）
- 信用卡詐騙偵測（是/不是詐騙）
- 疾病風險預測（有/沒有罹病風險）
- 客戶流失預測（會/不會流失）

🗣️ **白話說明：** 邏輯迴歸就像幫一件事情做「機率打分」——例如「這封郵件是垃圾的機率是 0.92」。得到機率後再設定一個門檻（通常是 50%），超過就判「是」、低於就判「否」。名字雖叫「迴歸」，但其實它做的是分類的事——這是史上最惡名昭彰的命名事故之一。

**Quick check：** 邏輯迴歸和線性迴歸的根本差異是什麼？
→ 線性迴歸預測連續值（迴歸任務）；邏輯迴歸預測類別（分類任務），名字有迴歸但任務是分類。

### 3.3 決策樹（Decision Tree）

**先懂一句話：** 用一系列「是/否問題」把資料層層分類，像是問答式的流程圖。

**概念說明：**
決策樹通過對資料集反複問問題來分類。例如「坪數 > 30 坪嗎？→ 是 → 建築年齡 < 10 年嗎？→ …」這樣一路向下直到決定結論。分裂時用**Gini 不純度（Gini Impurity）**或**資訊增益（Information Gain）**來判斷「哪個問題最能有效區分資料」——初級只要知道這兩個名字是「決策樹選擇分裂條件的標準」即可。

| 面向 | 說明 |
|---|---|
| 適用任務 | 分類（也可做迴歸） |
| 優點 | 高度可解釋（像流程圖，人可讀懂）、無需資料標準化 |
| 缺點 | **容易過擬合（overfitting）** 🔥 — 樹長太深就背記答案 |
| 資料需求 | 小到中型均可 |
| 解釋性 | ⭐⭐⭐⭐⭐（最高，可直接閱讀每個分支規則） |

**典型用途（Taiwan 情境）：**
- 銀行信貸審查（有規則且需解釋）
- 醫療診斷輔助流程
- 客戶分類規則

🗣️ **白話說明：** 決策樹就像「20 個問題猜謎遊戲」——你一路問是/否問題來猜出答案。缺點是如果問題太多、太細，就等於把訓練資料「背下來」，遇到新資料反而猜不準，這就是過擬合。

**Quick check：** 決策樹最大的弱點是什麼？ → 容易過擬合（樹越深、訓練準確率越高，但測試集表現可能很差）。

### 3.4 隨機森林（Random Forest）

**先懂一句話：** 建很多棵決策樹，讓它們「投票」決定答案，比單棵樹準確且穩定。

**概念說明：**
隨機森林是**集成學習（Ensemble Learning）**的代表，採用 **Bagging（Bootstrap Aggregating）**策略：
1. 從訓練資料中隨機抽樣（有放回），建出多棵決策樹
2. 每棵樹都只用隨機選取的一部分特徵來分裂
3. 預測時讓所有樹投票，多數決作為最終答案

「Bagging」概念＝**建多個模型、每個模型訓練在不同的隨機子集上，再聚合結果**。

| 面向 | 說明 |
|---|---|
| 適用任務 | 分類（也可做迴歸） |
| 優點 | 比單棵決策樹更準確、**大幅降低過擬合風險** 🔥、對離群值較穩健 |
| 缺點 | 可解釋性低於單棵決策樹（黑盒子程度提升）、訓練時間較長 |
| 資料需求 | 中型以上資料更穩定 |
| 解釋性 | ⭐⭐（整體森林難以直接閱讀）|

**典型用途（Taiwan 情境）：**
- 信用評分、詐騙偵測（比決策樹更準）
- 電商顧客流失預測 / 商品購買機率預測
- 醫療診斷（需要高準確率）

🗣️ **白話說明：** 一棵決策樹容易犯錯，就像只問一個人的意見。隨機森林等於同時問一千個人（每個人只看到隨機的部分資訊，避免大家都犯同樣錯誤），最後多數決——群眾的智慧比單一專家可靠。

**Quick check：** 隨機森林用的是 Bagging 還是 Boosting？ → **Bagging**（並行建多棵樹再投票）。Boosting 是另一個概念（見 3.6）。

### 3.5 支援向量機（SVM，Support Vector Machine）

**先懂一句話：** 找一條「離兩類資料點都最遠的邊界線」，讓分類邊界最清晰。

**概念說明：**
SVM（Support Vector Machine，支援向量機）在資料之間找一個**超平面（Hyperplane）**，使得這個超平面到兩邊最近資料點的「邊界距離（Margin）」最大化——這叫做**最大邊界（Maximum Margin）**。初級只需理解三個關鍵詞：超平面、邊界、支援向量（離邊界最近的那幾個點）。

對於線性不可分的資料，SVM 使用**核技巧（Kernel Trick）**把資料投射到更高維空間，讓原本分不開的資料變得可以被超平面切分——初級知道「核技巧讓 SVM 能處理非線性資料」即可。

| 面向 | 說明 |
|---|---|
| 適用任務 | 分類（也可做迴歸） |
| 優點 | 在高維資料（文字、圖像特徵）表現好；資料量小時仍有效 |
| 缺點 | 大型資料集訓練慢；對參數選擇敏感 |
| 資料需求 | **小到中型時 SVM 表現佳**；大型資料考慮其他方法 |
| 解釋性 | ⭐⭐（邊界概念直覺，但高維時難以視覺化） |

**典型用途（Taiwan 情境）：**
- 文字分類（小資料集的新聞分類）
- 影像分類（早期電腦視覺）
- 生物資訊（基因序列分類）

🗣️ **白話說明：** 把兩類點畫在紙上，SVM 找一條「和兩邊都隔最遠」的分界線——就像把兩隊人馬分開、中間留最寬的走廊。走廊越寬，以後遇到新資料分錯的機會越小。

**Quick check：** SVM 在什麼情況下特別適合？ → 資料量小到中型、高維特徵空間（例如文字資料）。

### 3.6 K 近鄰演算法（KNN，K-Nearest Neighbors）

**先懂一句話：** 「你和誰最近，你就是誰」——根據最近鄰居的類別投票決定新資料的類別。

**概念說明：**
KNN（K-Nearest Neighbors，K 近鄰演算法）的邏輯極其簡單：
1. 計算新資料點與所有訓練資料的「距離」（常用歐氏距離）
2. 找出最近的 K 個鄰居
3. 這 K 個鄰居中哪個類別最多，就把新資料歸入那個類別

KNN 有一個特殊性質：**懶惰學習（Lazy Learning）**——它在「訓練」時什麼都不做，只是把資料存起來；等到要預測時才開始計算距離。這和其他演算法（積極學習，Eager Learning）「先建模型再預測」的方式相反。

| 面向 | 說明 |
|---|---|
| 適用任務 | 分類（也可做迴歸） |
| 優點 | 概念簡單、不需訓練時間、能處理非線性邊界 |
| 缺點 | 預測時計算量大（資料大時很慢）；對特徵縮放（scale）敏感；高維時效果差（維度詛咒） |
| 資料需求 | **資料量小、維度低時效果好**；大資料不適合 |
| 解釋性 | ⭐⭐⭐（可以直接看「是因為和哪些鄰居相似」）|

**典型用途（Taiwan 情境）：**
- 推薦系統（「跟你口味相似的人也喜歡 X」）
- 小資料集的分類（醫療、金融小樣本）

🗣️ **白話說明：** 你搬到一個新社區，想知道這個社區的風氣——KNN 的做法是「問最近的 5 個鄰居（K=5），如果 4 個是安靜書香家庭，1 個是嘈雜派對咖，那你就歸類為安靜社區」。不需要事先建什麼模型，就是現查現用。

**Quick check：** KNN 是「懶惰學習」還是「積極學習」？ → **懶惰學習**——訓練時不建模型，預測時才計算距離。

### 3.7 梯度提升 / XGBoost（Gradient Boosting / XGBoost）

**先懂一句話：** 每次建一棵樹來「專門修補前一棵樹的錯誤」，一步一步把整體越做越好。

**概念說明：**
梯度提升（Gradient Boosting）屬於**集成學習**，但策略和隨機森林的 Bagging 不同——它用的是 **Boosting**：

| | Bagging（隨機森林）| Boosting（梯度提升）|
|---|---|---|
| 建樹方式 | 並行建多棵樹（互相獨立） | **依序**建樹（每棵樹修補前棵的錯誤） |
| 目的 | 降低變異（Variance） | 降低偏差（Bias） |
| 代表演算法 | 隨機森林 | Gradient Boosting、**XGBoost**、LightGBM |

**XGBoost（eXtreme Gradient Boosting）** 是梯度提升的高效實作版本，是許多資料科學競賽的冠軍神器，但初級只需知道它的概念定位：「Boosting 系的集成方法」。

| 面向 | 說明 |
|---|---|
| 適用任務 | 分類（也可做迴歸） |
| 優點 | 通常比隨機森林更高準確率；對不規則資料有很強泛化能力 |
| 缺點 | 參數多、調參複雜；訓練速度比隨機森林慢（依序而非並行）|
| 資料需求 | 中型到大型資料上閃耀 |
| 解釋性 | ⭐（更接近黑盒子）|

🗣️ **白話說明：** 想像一個班上有一百個學生在練習解數學題。Bagging（隨機森林）是：讓每個學生都去解不同版本的練習卷（各自為政），最後投票。Boosting 是：第一個學生解完後，第二個學生**專門去解第一個解錯的題目**，第三個再去解前兩個都解錯的……一輪一輪集中火力補洞。

**Quick check：** 隨機森林用 Bagging，梯度提升用 Boosting——兩者核心差異是什麼？
→ Bagging 是**並行**建多棵樹、互相獨立；Boosting 是**依序**建樹、每棵樹修補前棵的錯誤。

### 3.8 K-means 分群演算法

**先懂一句話：** 把沒有標籤的資料分成 K 個群，讓每個群內的資料點盡量靠近彼此。

**概念說明：**
K-means 是最常見的非監督式分群演算法。流程概念：
1. 隨機選 K 個「質心（Centroid）」
2. 把每個資料點分配到「最近的質心」那個群
3. 重新計算每個群的質心（群內所有點的平均位置）
4. 重複步驟 2–3，直到質心不再移動

關鍵特性：**需要預先指定 K（要分成幾群）**。如果不知道 K 應該是多少，可以用**手肘法（Elbow Method）**：試試 K=2, 3, 4, 5… 畫出每個 K 對應的「群內誤差」，找到曲線「彎折最明顯的那個點」（像手肘）就是合理的 K 值。

| 面向 | 說明 |
|---|---|
| 適用任務 | **分群**（非監督式）|
| 優點 | 簡單快速、可擴展到大資料 |
| 缺點 | 需要事先指定 K；對初始質心位置敏感；假設群是球形的 |
| 資料需求 | 各種規模均可，但高維效果下降 |
| 解釋性 | ⭐⭐⭐（可視化群中心）|

**典型用途（Taiwan 情境）：**
- 電商顧客分群（找出高價值/潛在/流失中的族群）
- 新聞主題分群（自動找出主題，無預設類別標籤）
- 市場區隔

🗣️ **白話說明：** 想像你有一百顆糖果要分成幾堆顏色相近的，K-means 就是不斷移動「每堆的中心點」直到每顆糖果都在離自己最近的那堆，直到結果穩定為止。你需要事先說好「要分幾堆」（K=?），它才知道如何開始。

**Quick check：** K-means 和 KNN 最根本的差異是什麼？
→ K-means 是**非監督式、分群**（無標籤，自己找群）；KNN 是**監督式、分類**（有標籤，看鄰居投票）。

---

### 3.9 階層式分群（Hierarchical Clustering）

**先懂一句話：** 不需要事先決定要分幾群，而是建出一棵「樹狀分群圖」，讓你事後再決定切在哪一層。

**概念說明：**
階層式分群（Hierarchical Clustering）不需要預先指定 K，而是建構一個叫做**樹狀圖（Dendrogram）**的樹狀結構——從「每個資料點都是獨立的群」開始，一步步把最相似的群合并（凝聚法），直到所有資料點都合成一群。

從 Dendrogram 上橫切一刀（選擇切割高度），就可以得到你想要的群數——非常有彈性。

| 面向 | 說明 |
|---|---|
| 適用任務 | **分群**（非監督式）|
| 優點 | **不需要預先指定 K**；可視化好（Dendrogram 直觀）|
| 缺點 | 資料量大時計算成本高、速度慢，不適合百萬筆以上的大型資料集 |
| 資料需求 | 小到中型資料 |
| 解釋性 | ⭐⭐⭐⭐（Dendrogram 可視化直觀）|

🗣️ **白話說明：** K-means 像先宣布「今天分三組」再去分；階層式分群像「先讓大家自由結伴，最像的先配對，再把各對繼續合并，最後畫成一棵族譜樹」——然後你在族譜上「剪一刀」，就得到你要的分組數。

**Quick check：** 階層式分群與 K-means 的主要差異？
→ 階層式分群**不需要預先指定 K**，K-means **必須預先指定 K**。

---

### 3.10 主成分分析（PCA，Principal Component Analysis）

**先懂一句話：** 把高維資料「壓縮」到低維，保留最多的資訊，丟掉最少重要的部分。

**概念說明：**
PCA（Principal Component Analysis，主成分分析）是最常見的降維技術。它找出資料中「變化最大的方向」（稱為主成分，Principal Components），然後只保留前幾個最重要的方向，把其餘維度丟掉——這樣就把高維資料壓縮成低維，但保留了大部分資訊。

**解釋方差比（Explained Variance Ratio）**：每個主成分「解釋了原始資料幾%的變化」。初級只需知道：前幾個主成分的解釋方差比加起來越高，代表降維後保留的資訊越多。

| 面向 | 說明 |
|---|---|
| 適用任務 | **降維**（非監督式）|
| 優點 | 有效去除冗餘特徵、降低訓練複雜度、方便視覺化 |
| 缺點 | 降維後的主成分難以解釋（不再是原始特徵）；是線性方法 |
| 資料需求 | 各種規模均可 |
| 解釋性 | ⭐（轉換後的維度難以理解）|

**典型用途（Taiwan 情境）：**
- 把高維基因資料壓縮後再做分析
- 把高維特徵壓縮後加速模型訓練
- 資料視覺化（壓到 2D 畫散佈圖）

🗣️ **白話說明：** 想像有一張 100 頁的試卷，PCA 幫你找出「最重要的 5 頁」，把這 5 頁的精華捏成一份 5 頁摘要，讓你以後讀這份摘要就掌握大部分內容，不用翻全本。

**Quick check：** PCA 是監督式還是非監督式？輸出是類別還是新的座標？
→ 非監督式；輸出是「新的座標（主成分）」，不是原始特徵，也不是類別。

---

### 3.11 神經網路基礎

**先懂一句話：** 模仿生物神經元的連接方式，讓資料一層一層轉換，學出越來越抽象的特徵。

#### 感知機 / 神經元基本概念

最基本的神經元（感知機，Perceptron）接受多個輸入，每個輸入乘以一個「權重（Weight）」，加總後通過一個**激活函數（Activation Function）**，輸出結果。多個神經元層疊就形成神經網路。

#### 前向傳播 vs 反向傳播

| 步驟 | 名稱 | 說明（概念即可）|
|---|---|---|
| 1 | **前向傳播（Forward Propagation）** | 資料從輸入層→隱藏層→輸出層，計算出預測值 |
| 2 | 計算誤差 | 比較預測值與正確答案的差距（損失） |
| 3 | **反向傳播（Backpropagation）** | 把誤差「從輸出層往回傳」，調整每個權重 |
| 4 | 重複 | 不斷循環直到模型收斂 |

> ⚠️ 初級只需知道「前向傳播 = 計算預測、反向傳播 = 調整參數」的方向概念，**不需要了解導數推導或梯度下降數學**。

#### 激活函數（Activation Functions）

激活函數的作用是「在神經元的輸出加入非線性」，讓神經網路能學習非線性的複雜模式。

| 函數名稱 | 常用位置 | 白話說明 |
|---|---|---|
| **ReLU**（Rectified Linear Unit，線性整流函數）| 深層神經網路隱藏層 | 負數輸出 0，正數直接通過。最常用、訓練快 |
| **Sigmoid（S 型函數）** | 二元分類輸出層 | 把數值壓縮到 0–1（機率）。也用在邏輯迴歸 |
| **Softmax** | 多類別分類輸出層 | 把多個數值轉換成「各類別的機率（加起來=1）」|

🗣️ **白話說明：** 激活函數像「守門員」——控制每個神經元有多少訊號通過。沒有激活函數，再多層的神經網路都只能做線性運算，等於白搭；加了激活函數，才能學出「貓」「狗」「橘子」這種複雜的非線性邊界。

**Quick check：** ReLU 最常用在哪裡？Softmax 用在什麼任務？
→ ReLU：隱藏層（加速訓練）；Softmax：多類別分類的輸出層。

---

### 3.12 模型選擇原則

**資料量考量：**

| 資料量 | 推薦方向 |
|---|---|
| 非常小（< 1,000 筆）| 邏輯迴歸、SVM、KNN（特徵維度低且已縮放時）|
| 小到中（1,000–50,000 筆）| 決策樹、隨機森林、SVM（高維文字特徵）、Gradient Boosting |
| 大型（> 100,000 筆）| 隨機森林、XGBoost、神經網路 / DL |

**任務類型選擇：**

| 任務 | 首選方向 |
|---|---|
| 預測連續數值 | 線性迴歸 |
| 二元分類（需可解釋）| 邏輯迴歸、決策樹 |
| 分類（追求高精度）| 隨機森林、Gradient Boosting |
| 分群（無標籤）| K-means、階層式分群 |
| 降維 | PCA |
| 影像分類 | CNN（深度學習）|

**解釋性 vs 準確度取捨（金融 / 醫療特別重要）：**

```
解釋性高 ◄─────────────────────────► 準確度高
  線性迴歸  邏輯迴歸  決策樹  隨機森林  XGBoost  深度神經網路
```

## Section 4 · 易混淆概念對照表

### 對照 1：決策樹 vs 隨機森林

| 面向 | 決策樹（Decision Tree）| 隨機森林（Random Forest）|
|---|---|---|
| 數量 | 1 棵樹 | **多棵樹**（通常數百棵）|
| 方法 | 單獨決策 | **Bagging**（集成學習）|
| 過擬合風險 | **高**（容易過擬合） | **低**（多棵樹平均降低風險）|
| 可解釋性 | **高**（可讀每個分支）| 低（整體難以直接閱讀）|
| 準確度 | 一般 | **通常更高** |
| 訓練速度 | 快 | 較慢（但可並行）|

**考題判斷：** 看到「集成學習 + 並行建多棵樹 + 投票」→ **隨機森林（Bagging）**。看到「單棵樹、高解釋性但易過擬合」→ **決策樹**。

---

### 對照 2：SVM vs KNN

| 面向 | SVM（支援向量機）| KNN（K 近鄰）|
|---|---|---|
| 學習策略 | **積極學習**（訓練時建模型）| **懶惰學習**（訓練時只存資料）|
| 預測速度 | 快（模型已建好）| 慢（預測時才計算距離）|
| 最佳資料規模 | **小到中型**（高維特徵）| **小到中型**（低維特徵）|
| 核心概念 | 最大邊界、超平面 | 距離度量、K 個鄰居投票 |
| 解釋性 | ⭐⭐ | ⭐⭐⭐ |

**考題判斷：** 看到「懶惰學習」或「訓練時不建模型」→ **KNN**。看到「最大邊界」或「超平面」→ **SVM**。

---

### 對照 3：K-means vs KNN（最高頻混淆！）

| 面向 | K-means | KNN（K-Nearest Neighbors，K 近鄰演算法）|
|---|---|---|
| 學習典範 | **非監督式** | **監督式** |
| 任務 | **分群** | **分類**（也可做迴歸）|
| 有無標籤 | **不需要** | **需要** |
| K 代表什麼 | 「要分成幾**群**」| 「看最近的幾個**鄰居**」|
| 輸出 | 群號（cluster ID）| 類別（class label）|
| 記憶方式 | K 均值 → **分群** | K 近鄰 → **分類** |

**考題判斷：** 「下列何者為非監督式學習？A. KNN　B. K-means」→ **B（K-means）**。

---

### 對照 4：Bagging vs Boosting（集成學習兩種策略）

| 面向 | Bagging | Boosting |
|---|---|---|
| 建樹順序 | **並行**（同時建多棵樹）| **依序**（每棵修補前棵的錯誤）|
| 目的 | 降低**變異**（防止過擬合）| 降低**偏差**（提升準確度）|
| 代表演算法 | **隨機森林** | **Gradient Boosting、XGBoost** |
| 過擬合風險 | 低 | 較高（若樹太多）|

**考題判斷：** 「隨機森林使用哪種集成策略？」→ **Bagging**。「XGBoost 使用哪種集成策略？」→ **Boosting**。

---

### 對照 5：線性迴歸 vs 邏輯迴歸（命名陷阱！）

| 面向 | 線性迴歸（Linear Regression）| 邏輯迴歸（Logistic Regression）|
|---|---|---|
| 任務 | **迴歸**（預測連續值）| **分類**（預測類別）🔥 |
| 輸出 | 連續數值（-∞ 到 +∞）| 機率（0 到 1）→ 類別 |
| 核心概念 | 最小平方法、擬合直線 | Sigmoid 函數、機率分界 |
| 典型例子 | 房價預測、銷售額預測 | 垃圾郵件、詐騙偵測 |

**考題判斷：** 「邏輯迴歸屬於分類還是迴歸演算法？」→ **分類（雖名字有迴歸，但任務是分類）**。

---

## Section 5 · 口訣

### 監督式七模型 ── 「線邏樹森機鄰升」

```
  線 = 線性迴歸   （Linear Regression）   ── 預測數值
  邏 = 邏輯迴歸   （Logistic Regression） ── 分類（名字騙人！）
  樹 = 決策樹     （Decision Tree）       ── 易過擬合
  森 = 隨機森林   （Random Forest）       ── Bagging 集成
  機 = SVM       （Support Vector Machine）── 最大邊界
  鄰 = KNN       （K-Nearest Neighbors） ── 懶惰學習
  升 = 梯度提升   （Gradient Boosting）   ── Boosting 集成
```

聯想句：「**線邏樹、森機鄰升**」——七種監督式學習模型，大致從簡單可解釋模型（左），到較複雜、較難解釋的集成模型（右）。

---

### 非監督三模型 ── 「K群階降PCA」

```
  K群 = K-means              ── 需要指定 K，分成 K 個群
  階  = 階層式分群             ── 不需指定 K，建 Dendrogram
  PCA = 主成分分析             ── 降維、保留方差
```

聯想句：「**K 群找質心、階層畫樹枝、PCA 壓維度**」。

---

### Bagging vs Boosting ── 「平行投票是 Bag、依序補洞是 Boost」

```
  Bagging：並行  → 多棵樹同時訓練 → 投票  → 隨機森林
  Boosting：依序 → 一棵修補一棵   → 累加  → XGBoost
```

口訣：「**Bag 裝多顆球（並行），Boost 火箭升空（依序加速）**」。

---

### 激活函數三劍客 ── 「ReLU 隱層、Sigmoid 二元、Softmax 多類」

```
  ReLU    → 隱藏層（hidden layer）── 最常用，加速訓練
  Sigmoid → 二元分類輸出（是/否）
  Softmax → 多類別分類輸出（類別機率加總=1）
```

口訣：「**隱層 ReLU、二元 Sigmoid、多類 Softmax**」。

---

### KNN vs K-means 最終記憶鑰匙

```
  KNN（K-Nearest Neighbors，K 近鄰演算法）：
    K = 近鄰數 → 監督式 → 分類
  K-means：
    K = 群數   → 非監督式 → 分群
```

一句話：「**KNN 抄鄰居（監督），K-means 分堆堆（非監督）**」。

## Section 6 · 考試陷阱

**陷阱 1：邏輯迴歸是迴歸演算法**
❌「邏輯迴歸的任務是預測連續數值」
✅ 邏輯迴歸雖有「迴歸」二字，但任務是**分類**（輸出機率→類別）。這是史上最常出的命名陷阱。

---

**陷阱 2：KNN 和 K-means 是同類**
❌「KNN 和 K-means 都有 K，應該是同一類演算法」
✅ **KNN = 監督式分類（有標籤）；K-means = 非監督式分群（無標籤）**。K 代表的意義完全不同。

---

**陷阱 3：隨機森林使用 Boosting**
❌「隨機森林通過逐步修補弱模型來提升性能」
✅ 隨機森林使用的是 **Bagging**（並行建多棵樹再投票）。Boosting 是 Gradient Boosting / XGBoost 的策略。

---

**陷阱 4：KNN 是積極學習（需要建模型）**
❌「KNN 在訓練時建立複雜模型，預測很快」
✅ KNN 是**懶惰學習（Lazy Learning）**——訓練時什麼都不做，只存資料；**預測時才計算距離**，因此預測慢。

---

**陷阱 5：決策樹不會過擬合**
❌「決策樹結構清晰，不會有過擬合問題」
✅ 決策樹**極度容易過擬合**——樹越深就越「背答案」，訓練集 100% 但測試集糟糕。隨機森林正是為了解決這個問題而生。

---

**陷阱 6：K-means 不需要指定 K**
❌「K-means 會自動決定要分幾群」
✅ K-means **必須預先指定 K（要分幾群）**，這是它最大的限制之一。手肘法（Elbow Method）是幫你選 K 的工具，但仍需人工決定。

---

**陷阱 7：深度學習一定比傳統 ML 好**
❌「深度神經網路是最強的，所有問題都應該用 DL」
✅ 在**小資料量、結構化表格、需要可解釋性**的情境下，傳統 ML（決策樹、邏輯迴歸）往往更合適。深度學習需要大量資料、高運算資源，且解釋性差。

---

**陷阱 8：PCA 是監督式學習**
❌「PCA 用原始特徵預測目標值，所以是監督式」
✅ PCA 是**非監督式降維**——它不需要任何標籤，只分析資料本身的變異結構。

---

**陷阱 9：SVM 最適合大型資料集**
❌「SVM 是最強的演算法，什麼資料都適合」
✅ SVM 在**小到中型資料集、高維特徵**時效果好；大型資料集時訓練非常慢，應考慮隨機森林或 XGBoost。

---

**陷阱 10：反向傳播需要懂微積分才能理解**
❌（初級學生常以為這個要考計算）
✅ **初級只考概念方向**：反向傳播 = 「把誤差從輸出層往回傳，用來調整每個連接的權重」——**方向概念**即可，不考導數公式。

---

```
  │
  └─ 輸入有影像 / 語音 / 文字（大量）？
      └─ ──► 考慮深度學習（CNN / RNN / Transformer）
```

### 關鍵詞 → 模型對照表

| 題目關鍵詞 | 對應模型 |
|---|---|
| 預測房價 / 銷售額（連續值）| 線性迴歸 |
| 垃圾郵件 / 詐騙（是/否分類）| 邏輯迴歸 |
| 流程圖、規則、可解釋分類 | 決策樹 |
| 集成、Bagging、多棵樹投票 | 隨機森林 |
| Boosting、依序修補弱分類器 | Gradient Boosting / XGBoost |
| 懶惰學習、看鄰居投票 | KNN |
| 最大邊界、超平面、小資料分類 | SVM |
| 無標籤、分群、質心 | K-means |
| 無標籤、不指定 K、Dendrogram | 階層式分群 |
| 降維、解釋方差比 | PCA |
| 影像辨識、卷積 | CNN |
| 序列資料、時間序列、文字 | RNN / LSTM |
| 隱藏層、激活函數 | 神經網路 |

### 情境練習（3 題）

**情境 A：** 某銀行有 10 萬筆信用卡申請，每筆都標記了「核准/拒絕」。銀行希望訓練一個模型，同時要求能向監管單位解釋「為何拒絕某申請人」。最適合的模型是？

→ **答：邏輯迴歸或決策樹**（有標籤 → 監督式；輸出類別 → 分類；需要高解釋性 → 邏輯迴歸/決策樹比隨機森林或深度學習更合適）

**情境 B：** 某零售商有 50 萬名顧客，但沒有任何「顧客類型」標籤，想依購買行為分成若干群體以利精準行銷。最適合的模型是？

→ **答：K-means**（無標籤 → 非監督式；分群目的；資料量大且 K-means 可擴展）

**情境 C：** 某工廠有大量不良品照片（已標記良品/不良品），希望用影像辨識自動檢測瑕疵。最適合的技術方向是？

→ **答：CNN（卷積神經網路）**（有標籤影像 → 監督式分類；影像資料 → CNN）
