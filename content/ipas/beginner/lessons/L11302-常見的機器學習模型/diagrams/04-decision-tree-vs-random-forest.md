# Diagram 04 — Decision Tree vs Random Forest（決策樹 vs 隨機森林）

> Source: L11302 Section 3.3–3.4 + Section 4 · 對照 1

## 單棵決策樹 vs 隨機森林

```
單棵決策樹（Decision Tree）：
─────────────────────────────
         坪數 > 30?
        /         \
      是             否
      |               |
   地段好?         屋齡 < 10年?
   /      \          /       \
 是        否      是           否
 |          |      |             |
高價值      中等   中等          低價值
（預測完成）

⚠️ 問題：樹越深 = 越容易「背答案」= 過擬合


隨機森林（Random Forest，Bagging 策略）：
────────────────────────────────────────
  隨機子集 A         隨機子集 B         隨機子集 C
  （部分資料）       （部分資料）       （部分資料）
       ↓                  ↓                  ↓
  決策樹 #1          決策樹 #2          決策樹 #3
  預測：高價值       預測：高價值       預測：中等
       ↓                  ↓                  ↓
           ─────── 多數投票 ───────
                    ↓
              最終預測：高價值（2:1 投票）

✅ 好處：多棵樹平均，誤差抵銷，不易過擬合
```

## 關鍵比較表

| 面向 | 決策樹 | 隨機森林 |
|---|---|---|
| 樹的數量 | **1 棵** | **多棵**（通常數百棵） |
| 集成策略 | 無 | **Bagging**（自助聚合）|
| 訓練方式 | 在全部資料上訓練 | 每棵樹在隨機子集（Bootstrap sample）上訓練 |
| 特徵選取 | 用全部特徵 | 每次分裂隨機選部分特徵 |
| 過擬合風險 | **高**（樹深時） | **低**（多棵樹平均效果）|
| 可解釋性 | **高**（可視覺化整棵樹）| **低**（整體難以直接閱讀）|
| 準確度 | 一般 | **通常更高** |
| 訓練速度 | 快 | 較慢（但各樹可並行）|

## Bagging 概念圖

```
原始訓練資料集（N 筆）
        │
        ├─ 隨機抽樣（有放回）→ 子集 1 → 決策樹 1 → 預測 1
        ├─ 隨機抽樣（有放回）→ 子集 2 → 決策樹 2 → 預測 2
        ├─ 隨機抽樣（有放回）→ 子集 3 → 決策樹 3 → 預測 3
        │             ...
        └─ 隨機抽樣（有放回）→ 子集 K → 決策樹 K → 預測 K
                                                        │
                                            投票 / 平均 ┘
                                                ↓
                                         最終預測結果
```

> Bagging = Bootstrap Aggregating
> Bootstrap（自助法）= 有放回抽樣，讓每棵樹見到不同版本的資料

## 考試判斷

- 「集成學習 + 並行建多棵樹 + 投票」→ **隨機森林（Bagging）**
- 「依序建樹、每棵修補前棵錯誤」→ **Gradient Boosting / XGBoost（Boosting）**
- 「隨機森林使用的是什麼策略？」→ **Bagging，不是 Boosting**
