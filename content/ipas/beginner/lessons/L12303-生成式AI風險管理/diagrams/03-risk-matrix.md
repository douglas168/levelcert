## 標準 2×2 風險矩陣

```
                        衝擊 Impact（發生後損失有多大）
                        低 (Low)              高 (High)
                   ┌─────────────────┬────────────────────┐
               高  │                 │                    │
               H  │  🟡 Accept+SOP  │  🔴 Mitigate       │
               i  │  ────────────── │  ──────────────    │
               g  │  高可能 × 低衝擊 │  高可能 × 高衝擊   │
               h  │                 │                    │
  可能性            │  策略：例行化    │  策略：最優先緩解   │
  Likelihood        │  對策：SOP+快速 │  對策：HITL+         │
               ↑  │  應對流程       │  Guardrails+         │
               │  │                 │  Red Team           │
               ├──├─────────────────┼────────────────────┤
               │  │                 │                    │
               低  │  🟢 Accept      │  🟠 Transfer       │
               L  │  ────────────── │  ──────────────    │
               o  │  低可能 × 低衝擊 │  低可能 × 高衝擊   │
               w  │                 │                    │
                  │  策略：列冊觀察  │  策略：保險/合約    │
                  │  對策：記錄不緊  │  對策：資安保險+     │
                  │  急處理         │  多因子驗證+         │
                  │                 │  應變計畫           │
                  └─────────────────┴────────────────────┘
                       低 (Low) ←── 可能性 Likelihood ──► 高 (High)
```

---

## 四個策略動詞（口訣：降轉接避）

| 象限位置 | 策略 | 英文 | 核心動作 |
|---|---|---|---|
| 高可能 × 高衝擊 | **降低** | Mitigate | 多層防線（HITL + Guardrails + Red Team）|
| 低可能 × 高衝擊 | **轉移** | Transfer | 保險/合約移轉責任 + 應變計畫 |
| 高可能 × 低衝擊 | **接受+SOP** | Accept + SOP | 標準作業流程化，快速處理 |
| 低可能 × 低衝擊 | **接受** | Accept | 列冊觀察，不緊急處理 |
| 極端無法承擔 | **避免** | Avoid | 退出/不做此項目 |

## 考試最高頻陷阱

### 陷阱 1：低可能×高衝擊 → 選 Accept？

```
題目情境：深偽詐騙事件發生可能性低，但一旦發生損失極大

❌ 錯誤答案：「可能性低就接受（Accept）它」
✅ 正確答案：「低可能×高衝擊 → Transfer 轉移（保險/合約）+ 應變計畫」
```

### 陷阱 2：「避免（Avoid）」不是「不處理」

```
Avoid（避免）= 乾脆不做這個項目/功能（退出）
Accept（接受）= 做但知道有風險，選擇接受

極端例子：某應用在法律上完全無法合規 → Avoid（放棄導入）
一般低風險：偶爾輸出格式怪 → Accept（不影響業務就接受）
```

### 陷阱 3：高高象限一定要 Mitigate，不能 Avoid？

```
Mitigate = 加防線繼續做（大部分高高情境）
Avoid = 這個功能無論怎麼加防線都無法達到合規要求 → 不做

區分關鍵：「加防線能讓它達到可接受水準嗎？」
能 → Mitigate（降低）
不能 → Avoid（避免）
```

## 快速記憶圖

```
                 衝擊
                  │
             高   │  Transfer 轉 │ Mitigate 降
                  │  （保險計畫）│（最優先！）
                  ├─────────────┼─────────────
             低   │  Accept 接  │ Accept+SOP 接
                  │  （列冊觀察）│（例行化）
                  └─────────────┴─────────────
                 低             高
                          可能性
```

> 口訣：**降轉接避**
> - 高高 = **降**（最優先緩解）
> - 低高 = **轉**（保險轉移）
> - 高低 = **接**+SOP（例行化）
> - 低低 = **接**（觀察）
> - 極端 = **避**（退出）
