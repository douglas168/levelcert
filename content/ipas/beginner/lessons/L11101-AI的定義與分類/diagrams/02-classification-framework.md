# AI 分類框架總覽

三種分類法各自從不同角度描述同一個 AI 系統。

```
┌──────────────────────────────────────────────────────────────────────────┐
│              AI（Artificial Intelligence，人工智慧）分類系統               │
├──────────────────────┬──────────────────────┬───────────────────────────┤
│  按能力（Capability） │  按功能（Function）   │  按哲學（Philosophy）     │
│  能做「多廣的事」？   │  「如何處理資訊」？   │  有沒有「真正的理解」？   │
│  提出者：學術慣例     │  提出者：Hintze       │  提出者：John Searle      │
├──────────────────────┼──────────────────────┼───────────────────────────┤
│                      │                      │                           │
│  ASI                 │  Type 4              │                           │
│  超人工智慧          │  Self-Aware          │  Strong AI（強人工智慧）   │
│  ❌ 純粹理論          │  自我意識             │  真正擁有意識與理解力      │
│  超越所有人類智慧     │  ❌ 純粹理論          │  ❌ 目前不存在             │
│                      │                      │                           │
│  AGI                 │  Type 3              │                           │
│  通用人工智慧         │  Theory of Mind      │                           │
│  ❌ 理論階段          │  心智理論             │                           │
│  跨領域通用智慧       │  ❌ 研究中            │                           │
│                      │  理解情緒/意圖        │                           │
│                      │                      │                           │
│  🔥 ANI              │  🔥 Type 2           │  🔥 Weak AI（弱人工智慧） │
│  弱人工智慧 / 窄 AI   │  Limited Memory      │  只是模擬智慧行為          │
│  ✅ 目前所有 AI       │  有限記憶             │  沒有真正的意識            │
│  只能處理特定任務     │  ✅ 目前大多數 AI     │  ✅ 目前所有 AI 都是弱 AI  │
│                      │  利用歷史資料決策     │                           │
│                      │                      │                           │
│                      │  Type 1              │                           │
│                      │  Reactive Machines   │                           │
│                      │  反應式機器           │                           │
│                      │  ✅ 已存在（早期）    │                           │
│                      │  只看當前輸入         │                           │
│                      │  IBM Deep Blue 為例  │                           │
│                      │                      │                           │
└──────────────────────┴──────────────────────┴───────────────────────────┘
```

## 跨框架對應範例

| AI 系統 | 能力分類 | 功能分類 | 哲學分類 |
|---------|---------|---------|---------|
| ChatGPT | ANI（弱/窄） | Limited Memory（有限記憶） | Weak AI（弱 AI） |
| IBM Deep Blue | ANI（弱/窄） | Reactive Machines（反應式） | Weak AI（弱 AI） |
| 科幻電影 AI | AGI（通用） | Theory of Mind / Self-Aware | Strong AI（強 AI）的概念 |

## ⚠️ 最常見混淆陷阱

```
❌ 錯誤對應：ANI = Weak AI（完全等同）
✅ 正確理解：
   - Weak/Strong → 問「有沒有意識」（Searle 哲學問題）
   - ANI/AGI/ASI → 問「能做多少任務」（能力範圍問題）
   - 現實中所有 AI 既是 ANI 也是 Weak AI，但這是兩個不同維度
```
