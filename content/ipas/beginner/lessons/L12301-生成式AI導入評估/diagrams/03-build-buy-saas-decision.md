# Build / Buy / SaaS 方案選擇決策樹

```
┌─────────────────────────────────────────────────────────────────┐
│               解決方案選擇決策框架（三選一）                        │
└─────────────────────────────────────────────────────────────────┘

              資料能上雲嗎？有法規限制嗎？
                      │
              ┌───────┴───────┐
              │  不能上雲/      │  可以上雲
              │  嚴格合規要求   │
              ▼               ▼
         ┌─────────┐    需要高度客製化嗎？
         │  BUILD  │         │
         │  自建    │   ┌─────┴─────┐
         └─────────┘   │  需要      │  不太需要
                       │  客製化     │  需求較標準
                       ▼            ▼
               需要快速上線嗎？  ┌─────────┐
                    │           │  SaaS   │
             ┌──────┴──────┐   │  訂閱制  │
             │  需要快速    │  └─────────┘
             │  上線         │
             ▼              ▼
        ┌─────────┐    ┌─────────┐
        │  SaaS   │    │   BUY   │
        │  訂閱制  │    │  購買授權 │
        └─────────┘    └─────────┘
```

## 三方案速記比較表

|  | SaaS（訂閱制） | Buy（購買授權） | Build（自建） |
|--|--------------|--------------|-------------|
| **上線速度** | ⚡ 最快（週） | 中（月） | 🐢 最慢（月～年） |
| **初期成本** | 💰 最低 | 中 | 💸 最高 |
| **客製程度** | 最低（標準） | 中 | 最高（完全掌控） |
| **資料主權** | 最低（存供應商） | 中 | 最高（私有部署） |
| **維護責任** | 供應商負責 | 共同負責 | 自行維護 |
| **適用情境** | 快速驗證、非核心業務 | 標準需求＋少量客製 | 法規限制、競爭差異化 |

## 記憶口訣

> **速度：SaaS 最快 → Buy 次之 → Build 最慢**
>
> **掌控：Build 最高 → Buy 次之 → SaaS 最低**

## 常見判斷情境

| 情境關鍵字 | 建議方案 |
|----------|---------|
| 無AI工程師 + 資金有限 + 快速上線 | SaaS |
| 標準需求 + 少量客製 + 合理期限 | Buy |
| 資料不能上雲 + 法規嚴格 + 核心業務 | Build |
| 需要競爭差異化 + 長期主導權 | Build |

> **考試陷阱**：Build 的優勢是「資料主權控制最高，可私有部署」，不是「最安全」（安全是架構設計問題）。
