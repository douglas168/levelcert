# 四大評估維度「技工模效」

```
┌─────────────────────────────────────────────────────────────────┐
│                    生成式AI導入評估四大維度                        │
│                       口訣：技工模效                              │
└─────────────────────────────────────────────────────────────────┘

        ┌──────────────────────┬──────────────────────┐
        │   【技】 技術評估      │   【工】 工具評估      │
        │  Technical           │  Tool                │
        ├──────────────────────┼──────────────────────┤
        │ 核心問題：做得到嗎？   │ 核心問題：用哪個工具？  │
        │                      │                      │
        │ • 基礎設施（算力/網路）│ • 供應商比較          │
        │ • 資料品質（量/格式）  │ • 功能適配度          │
        │ • API整合能力         │ • 安全性/合規性        │
        │                      │ • 使用者介面適配       │
        ├──────────────────────┼──────────────────────┤
        │   【模】 模型評估      │   【效】 效能評估      │
        │  Model               │  Performance         │
        ├──────────────────────┼──────────────────────┤
        │ 核心問題：哪個模型最適 │ 核心問題：有沒有效益？  │
        │ 合？                  │                      │
        │                      │ • 導入後 KPI 變化      │
        │ • 準確率測試          │ • ROI 計算            │
        │ • 幻覺風險評估        │ • 效率提升比例         │
        │ • 模型版本/更新政策   │ • 業務指標（NPS等）    │
        └──────────────────────┴──────────────────────┘

                     ▲ 導入前評估 ▲                  ▲ 導入後評估 ▲
                  技術 + 工具 + 模型                     效能
```

## 維度對比速查

| 維度 | 口訣字 | 核心問題 | 時機 | 常見陷阱 |
|------|--------|---------|------|---------|
| 技術評估 | 技 | 做得到嗎？ | 導入前 | 只看硬體，忽略資料品質 |
| 工具評估 | 工 | 用哪個工具？ | 導入前 | 只看功能，忽略安全合規 |
| 模型評估 | 模 | 哪個模型最適合？ | 導入前 | 只追求最新/最大的模型 |
| 效能評估 | 效 | 有沒有效益？ | **導入後** | 混淆導入前後的評估時機 |

> **考試陷阱**：效能評估是「導入後」才能進行的唯一維度，其餘三者在導入前進行。
