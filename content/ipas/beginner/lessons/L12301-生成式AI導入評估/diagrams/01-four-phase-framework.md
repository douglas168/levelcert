# 《AI導入指引》四階段框架（構方驗實）

```
┌─────────────────────────────────────────────────────────────────┐
│               《AI導入指引》AI導入四階段流程                      │
└─────────────────────────────────────────────────────────────────┘

  ┌────────────┐     ┌────────────┐     ┌────────────┐     ┌────────────┐
  │  第一階段   │     │  第二階段   │     │  第三階段   │     │  第四階段   │
  │   構想評估  │────▶│   方案設計  │────▶│  驗證/POC  │────▶│  實施/營運  │
  │  Ideation  │     │  Solution  │     │ Validation │     │Implementation│
  └────────────┘     └────────────┘     └────────────┘     └────────────┘
        │                  │                  │                   │
        ▼                  ▼                  ▼                   ▼
  • 痛點分析          • Build/Buy/SaaS    • POC 小規模測試    • 全面部署
  • 場景選定          • KPI 定義          • 成功標準設定      • 員工培訓
  • AI適配度評估      • 技術選型          • Exit Criteria    • KPI 追蹤
  • 資源盤點          • 成本估算          • 可行性確認        • 持續優化

        │                                      │
        │          ← 可回退到前一階段 →          │
        └──────────────────────────────────────┘

口訣：構 → 方 → 驗 → 實（構方驗實）
```

## 關鍵特性

| 特性 | 說明 |
|------|------|
| 循序漸進 | 必須依序執行，不可跳過驗證直接實施 |
| 可回退 | 後一階段發現問題，可有序退回前一階段調整 |
| 四大評估 | 每個階段都應進行對應的技工模效評估 |
| 早期驗證 | POC（第三階段）是防止大規模資源浪費的關鍵閘門 |

> **考試陷阱**：「競爭對手已上線，跳過POC直接實施」— 這是最違反框架精神的做法。
