## 各階段快速定位卡

```
┌──────────────────────────────────────────────────────────────────────┐
│  POC（概念驗證）                                                      │
│  關鍵詞：最小、技術可行、快速、便宜、單一場景                        │
│  目標：「這技術在我們場景中行得通嗎？」                              │
│  成功條件：技術可行 + 初步效益量化 + 資料品質確認                   │
├──────────────────────────────────────────────────────────────────────┤
│  Pilot（試行）                                                        │
│  關鍵詞：真實用戶、小規模、效益驗證、使用者回饋                     │
│  目標：「真實使用者覺得有效益嗎？」                                  │
│  成功條件：效益驗證 + 用戶回饋正向 + 資安合規確認                   │
├──────────────────────────────────────────────────────────────────────┤
│  Rollout（推廣）                                                      │
│  關鍵詞：跨部門、複製、標準化、SLA、維運                            │
│  目標：「能成功複製到多個團隊嗎？」                                  │
│  成功條件：跨部門導入完成 + SLA 達標 + 維運就緒                     │
├──────────────────────────────────────────────────────────────────────┤
│  Scale（規模化）                                                      │
│  關鍵詞：全組織、持續、ROI、外部整合、治理                          │
│  目標：「能在全組織持續產生價值嗎？」                                │
│  成功條件：全組織上線穩定 + ROI 達標 + 持續優化機制到位             │
└──────────────────────────────────────────────────────────────────────┘
```

## 常見考題陷阱——跳階問題

```
    ✗ 錯誤思維：
    POC 成功 ──────────────────────────────► Scale
    （跳過 Pilot 和 Rollout，直接大規模）

    ✓ 正確順序：
    POC ──► Pilot ──► Rollout ──► Scale
    （每個階段都有出口標準，通過才能進入下一階段）

    考試中若出現「POC 成功後直接全面部署」→ 選「錯誤」
    考試中若出現「跳過 Pilot 直接 Rollout」→ 選「錯誤」
```

## 考試重點

- **四個階段按順序**：POC → Pilot → Rollout → Scale，不可跳躍
- **技術風險**在 POC 最高，在 Scale 最低（方向相反於成熟度）
- **資源投入**隨階段遞增，Scale 是最大投入
- **出口標準**是每個階段的「通行證」，沒過就不能進入下一階段
- POC 成功**不代表**商業可行，Pilot 才是驗證商業效益
