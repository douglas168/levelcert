## HITL 的正確位置（事前設計，非應急措施）

```
    ┌─────────────────────────────────────────────────────────────────┐
    │                  HITL（Human-in-the-Loop，人機協作回圈）         │
    │                                                                  │
    │  ⚠ 重要：HITL 不是應急措施，是「事前設計」的審核機制            │
    │                                                                  │
    │  適用場景：                                                      │
    │  • AI 信心分數低於閾值時 → 自動觸發人工審核                     │
    │  • 高風險決策（醫療診斷、貸款審核、法律建議）                   │
    │  • 輸出結果影響範圍廣、不可逆的情況                             │
    │                                                                  │
    │  設計方式：在系統「架構層」預埋人工審核節點                     │
    │  ≠ 發生問題後臨時叫人來救火                                     │
    └─────────────────────────────────────────────────────────────────┘

    正常流程中的 HITL 設計示意：

    使用者輸入 → AI 處理 → 信心評估
                               │
                    ┌──────────┴──────────┐
                    │                     │
               高信心度               低信心度
                    │                     │
                    ▼                     ▼
              直接輸出              → 人工審核節點 →  輸出
              (自動化)              (HITL 觸發)    (人工確認後)
```

## 四種機制對比表

```
╔══════════════╦════════════════════════╦══════════════════╦══════════════════════╗
║   機制       ║   觸發條件             ║   動作           ║   恢復方式           ║
╠══════════════╬════════════════════════╬══════════════════╬══════════════════════╣
║ Fallback     ║ 主系統降級/失效        ║ 切換至備援系統   ║ 主系統修復後切回    ║
║ 備援         ║                        ║ 或人工流程       ║                      ║
╠══════════════╬════════════════════════╬══════════════════╬══════════════════════╣
║ Rollback     ║ 新版本有 bug/效能差    ║ 退回前一版本     ║ 修復新版後重新部署  ║
║ 回滾         ║                        ║                  ║                      ║
╠══════════════╬════════════════════════╬══════════════════╬══════════════════════╣
║ Kill Switch  ║ 重大安全/倫理問題      ║ 立即完全停止     ║ 全面稽核後才重啟    ║
║ 緊急關閉     ║                        ║ AI 功能          ║                      ║
╠══════════════╬════════════════════════╬══════════════════╬══════════════════════╣
║ HITL         ║ 事前設計的觸發條件     ║ 人工介入審核     ║ 持續機制，非一次性  ║
║ 人機協作回圈 ║ （信心低/高風險決策）  ║ 確認後繼續       ║                      ║
╚══════════════╩════════════════════════╩══════════════════╩══════════════════════╝
```

## 考試重點

- **Kill Switch** = 最嚴重、最後手段，停止整個 AI 功能
- **Rollback** = 退回「版本」，不是停止服務
- **Fallback** = 切換至「備援」（可能是舊系統或人工流程）
- **HITL** = 事前嵌入架構中，**不是**發生問題後的應急工具
- 嚴重程度排序：Kill Switch > Rollback > Fallback（可類比「火警逃生層級」）
