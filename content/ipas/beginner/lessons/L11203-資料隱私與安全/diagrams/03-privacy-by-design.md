# 圖 3：隱私設計（Privacy by Design）七大原則

```
  ┌────────────────────────────────────────────────────────────────────┐
  │           Privacy by Design（PbD）隱私設計七大原則                    │
  │              Ann Cavoukian 博士，1995年提出，GDPR 採納               │
  └────────────────────────────────────────────────────────────────────┘
            │
            ├─ 原則 1：主動預防，而非事後補救
            │   Proactive not Reactive; Preventative not Remedial
            │   ▶ 考試核心：在問題發生「之前」就設計進去，非事後修補
            │
            ├─ 原則 2：隱私作為預設值
            │   Privacy as the Default Setting
            │   ▶ 系統預設最嚴格的隱私保護，無需用戶額外設定
            │
            ├─ 原則 3：隱私內嵌於設計中
            │   Privacy Embedded into Design
            │   ▶ 隱私保護是系統架構的一部分，不是附加功能
            │
            ├─ 原則 4：全功能，非零和
            │   Full Functionality — Positive-Sum, not Zero-Sum
            │   ▶ 隱私與安全可以並存，不是非此即彼的取捨
            │
            ├─ 原則 5：全程安全保護
            │   End-to-End Security — Full Lifecycle Protection
            │   ▶ 從資料蒐集到刪除的完整生命週期都受保護
            │
            ├─ 原則 6：可見性與透明性
            │   Visibility and Transparency — Keep it Open
            │   ▶ 資料處理方式對外公開，可被查核
            │
            └─ 原則 7：尊重用戶隱私
                Respect for User Privacy — Keep it User-Centric
                ▶ 以用戶利益為中心，提供強大的隱私預設值
```

## 初級考試重點

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  考試最常考的概念：                                                  │
  │                                                                    │
  │  Q: 隱私設計（Privacy by Design）的核心精神是什麼？                  │
  │  A: 「主動預防而非事後補救」——在系統設計最初階段就將                 │
  │     隱私保護機制內嵌，不是等問題發生後才來處理。                    │
  │                                                                    │
  │  Q: Privacy by Design 和 GDPR 的關係？                             │
  │  A: GDPR 第25條「資料保護設計及預設保護（Data Protection             │
  │     by Design and by Default）」採納了 PbD 精神。                   │
  │                                                                    │
  │  ⚠️ 考試陷阱：「先推出產品再處理隱私問題」→ 違反 PbD 原則 1         │
  │  ⚠️ 考試陷阱：「隱私設計只適用於敏感資料」→ 錯，所有資料均適用      │
  └──────────────────────────────────────────────────────────────────┘
```

## 與個資法三行為的對應

| PbD 原則 | 對應台灣個資法精神 |
|---|---|
| 原則 1（主動預防） | 安全維護義務（第20-1條）— 未雨綢繆，非等外洩才處理 |
| 原則 2（隱私預設） | 第5條特定目的、比例原則 — 預設最小蒐集 |
| 原則 3（設計內嵌） | 第8條告知義務 — 蒐集前就設計好告知流程 |
| 原則 7（以用戶為中心） | 第3條當事人5大權利（查複補停刪）— 用戶主導自身個資 |
