```
  ┌──────────────────────────────────────────────────────────────────┐
  │  層次 1：傳輸安全                                                   │
  │  SSL/TLS（Secure Sockets Layer / Transport Layer Security）        │
  │  資料在網路傳輸時加密（HTTPS = HTTP + TLS）                         │
  │  保護：機密性（C） + 完整性（I）                                    │
  │                                                                    │
  │  ┌──────────────────────────────────────────────────────────────┐ │
  │  │  層次 2：靜態資料加密（Data at Rest Encryption）               │ │
  │  │  資料儲存在硬碟/資料庫時的加密                                  │ │
  │  │  保護：機密性（C）                                              │ │
  │  │                                                                │ │
  │  │  ┌──────────────────────────────────────────────────────────┐ │ │
  │  │  │  層次 3：存取控制（Access Control）                        │ │ │
  │  │  │  身分驗證（Authentication）+ 授權（Authorization）         │ │ │
  │  │  │  確保：只有授權人員能操作資料                               │ │ │
  │  │  │  保護：機密性（C） + 完整性（I）                            │ │ │
  │  │  │                                                            │ │ │
  │  │  │  ┌──────────────────────────────────────────────────────┐ │ │ │
  │  │  │  │  層次 4：去識別化（De-identification）                 │ │ │ │
  │  │  │  │  ├─ 假名化（Pseudonymization）                        │ │ │ │
  │  │  │  │  │   可逆，GDPR 下仍是個資，對照表另存                 │ │ │ │
  │  │  │  │  └─ 匿名化（Anonymization）                           │ │ │ │
  │  │  │  │      不可逆，對照表永久銷毀 → 脫離個資法               │ │ │ │
  │  │  │  └──────────────────────────────────────────────────────┘ │ │ │
  │  │  └──────────────────────────────────────────────────────────┘ │ │
  │  └──────────────────────────────────────────────────────────────┘ │
  └──────────────────────────────────────────────────────────────────┘
```

## CIA 三元素 ↔ DAD 威脅對應圖

```
  ┌─────────────────────────────────────────────────────────────────┐
  │                     CIA Triad（正向）                             │
  │                                                                   │
  │         C 機密性              I 完整性             A 可用性        │
  │       Confidentiality        Integrity           Availability     │
  │       （只有授權者能         （資料未被            （授權者需要     │
  │         看到資料）            未授權修改）           時用得到）     │
  │              │                    │                    │          │
  │              ▼                    ▼                    ▼          │
  │       D 洩漏                A 竄改               D 阻斷           │
  │       Disclosure            Alteration           Denial           │
  │       （駭客竊取）          （資料被偷改）        （DDoS攻擊）     │
  │                                                                   │
  │                     DAD Model（對應攻擊）                         │
  └─────────────────────────────────────────────────────────────────┘
```

## 情境判斷速查

| 事件描述 | CIA 受損項目 | DAD 攻擊類型 |
|---|---|---|
| 駭客竊取客戶資料庫並公開 | C 機密性 | D Disclosure（洩漏） |
| 勒索軟體加密資料庫，系統癱瘓 | A 可用性 | D Denial（阻斷） |
| 攻擊者修改帳單金額但未竊取 | I 完整性 | A Alteration（竄改） |
| DDoS 攻擊導致網站連續停擺 | A 可用性 | D Denial（阻斷） |
| 員工將個資傳至未加密郵件外洩 | C 機密性 | D Disclosure（洩漏） |
