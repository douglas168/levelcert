## Section 3 · 核心概念

### 3.1 機器學習是什麼？

#### 先懂一句話：
> ML（Machine Learning，機器學習）是讓電腦從**資料**中自動學出規律，而不是由人類一條條手寫規則。

| 比較面向 | 傳統程式設計 | 機器學習 |
|---|---|---|
| 輸入 | 規則 + 資料 | 資料 + 正確答案（標籤） |
| 輸出 | 答案 | 模型（= 學出來的規則） |
| 規則來源 | 人類工程師手寫 | 電腦從資料中自動學到 |
| 適合情境 | 規則明確、需 100% 正確 | 規則複雜、可接受機率性結果 |
| 典型例子 | 稅額計算、ATM 流程 | 垃圾信過濾、影像辨識 |

**口訣：傳統程式 = 規則 + 資料 → 答案；ML = 資料 + 答案 → 規則（模型）**

🗣️ **白話說明：** 教外國朋友認珍珠奶茶，你不可能寫一本「辨識珍奶規則書」——杯要透明、珍珠要黑、液體要棕色⋯⋯每季推新品就要全部重寫。比較有效的做法是給他看 500 張珍奶照片 + 500 張非珍奶照片，讓他自己從範例中「感覺出」珍奶的樣子。這就是 ML——**讓電腦從範例學規則，不是你把規則塞進它腦袋**。

**Quick check：**
- 「垃圾郵件過濾要用傳統程式還是 ML？」→ ML（規則會演化、寫不完）
- 「計算機的加法要用傳統程式還是 ML？」→ 傳統程式（規則明確、不能錯）

### 3.2 三大學習範式

#### 先懂一句話：
> ML 依照「有沒有標籤資料、如何學習」分為三大類：監督式、非監督式、強化學習。

#### 3.2.1 監督式學習（Supervised Learning，監督式學習）

- **學習方式：** 從「有標籤（Label）的資料」學習，讓模型學會「輸入 X → 輸出 Y」的對應關係
- **資料需求：** 需要大量**已標記**的訓練資料（成本高、耗人力）
- **兩個子任務：**

| 子任務 | 說明 | 例子 |
|---|---|---|
| 分類（Classification） | 預測**類別**（離散） | 垃圾信/非垃圾信、貓/狗、良品/瑕疵 |
| 回歸（Regression） | 預測**數值**（連續） | 房價預測、氣溫預測、業績預估 |

**代表情境：** 房價預測、人臉辨識、信用評分、醫療影像診斷

🗣️ **白話說明：** 就像你在學英文單字，老師給你一本「中文 + 英文對照」的字典（這是標籤資料），你從中學會「看到中文猜英文」的規律。監督式學習就是這樣——有「題目+答案」，讓模型學會做題目。

**Quick check：** 「預測 LINE 貼圖是否涉及版權侵害」是分類還是回歸？→ 分類（是/否 是離散類別）

---

#### 3.2.2 非監督式學習（Unsupervised Learning，非監督式學習）

- **學習方式：** 從**無標籤**的資料中自行發現隱藏結構或模式
- **資料需求：** 不需標籤（省成本，但解釋結果較難）
- **兩個子任務：**

| 子任務 | 說明 | 例子 |
|---|---|---|
| 分群（Clustering，分群） | 自動把相似的資料點歸成同一組 | 客戶分群、新聞文章聚類 |
| 降維（Dimensionality Reduction，降維） | 把高維資料壓縮成低維，保留重要結構 | 資料視覺化前處理、去除冗餘特徵 |

**代表情境：** 電商客群分析、異常偵測（Anomaly Detection）、推薦系統的前置分析

🗣️ **白話說明：** 想像你把一堆 Dcard 文章丟進去，但完全沒有分類標籤，模型自己去找「這些文章之間的相似性」，然後告訴你「這 300 篇都在講失戀、那 200 篇都在講求職」。沒有老師教，模型自己分組——這就是非監督式學習。

**Quick check：** 「把一萬位 LINE 用戶依使用行為分成 5 種族群」→ 非監督式（沒有預設標籤，自行分群）

---

#### 3.2.3 強化學習（Reinforcement Learning，強化學習）

- **學習方式：** 代理人（Agent）在環境（Environment）中透過**試錯**，依據獎勵（Reward）或懲罰來調整行為策略
- **資料需求：** 不需預先標記資料；從與環境的互動中產生資料
- **核心元素：** Agent、Environment、State（狀態）、Action（動作）、Reward（獎勵）

**代表情境：** 棋盤遊戲 AI（AlphaGo）、機器人控制、自動駕駛路線規劃、廣告競價策略

🗣️ **白話說明：** 就像訓練小狗——狗做對了就給零食（正向獎勵），做錯了就不給（負向）。狗慢慢學會什麼行為能獲得獎勵。強化學習的 AI「狗狗」就是 Agent，「房間」就是 Environment，零食就是 Reward。它透過不斷試錯來學會最佳策略。

**Quick check：** 「AI 玩電玩遊戲、靠得高分來改進策略」→ 強化學習

---

#### 三大範式快速比較

| | 監督式學習 | 非監督式學習 | 強化學習 |
|---|---|---|---|
| 標籤需求 | ✅ 需要標籤 | ❌ 不需標籤 | ❌ 不需預先標籤 |
| 學習信號 | 正確答案（標籤） | 資料內在結構 | 環境給的獎勵/懲罰 |
| 典型任務 | 分類、回歸 | 分群、降維 | 策略優化 |
| 考試關鍵詞 | 「有標籤」「分類」「預測」 | 「無標籤」「分群」「分析結構」 | 「試錯」「獎勵」「代理人」「策略」 |

### 3.3 機器學習流程

#### 先懂一句話：
> ML 專案有 6 個步驟：資料蒐集 → 特徵工程 → 訓練 → 驗證 → 測試 → 部署，是一個**循環**，不是直線。

參考：`diagrams/04-ml-workflow.md`

```
  資料蒐集 → 特徵工程 → 模型訓練 → 驗證調整 → 測試評估 → 部署監控
      ↑                                               |
      └─────────── 表現退化時，回頭重新訓練 ────────────┘
```

| 步驟 | 動作 | 關鍵說明 |
|---|---|---|
| 1. 資料蒐集（Data Collection） | 收集、清理、整理資料 | 品質 > 數量；garbage in → garbage out |
| 2. 特徵工程（Feature Engineering，特徵工程） | 選擇/萃取/轉換有用特徵 | 好的特徵能大幅提升模型表現 |
| 3. 模型訓練（Model Training） | 用訓練集讓模型學習 | 模型從資料中調整內部參數 |
| 4. 驗證（Validation） | 用驗證集調整超參數 | 比較不同模型、防止 Overfitting |
| 5. 測試（Test） | 用測試集做最終評估 | 只用一次；衡量泛化能力 |
| 6. 部署與監控（Deploy & Monitor） | 上線、追蹤效能 | 概念漂移（Concept Drift）需重訓 |

🔥 **考試重點：流程是循環（iterative），不是一次性直線。**

🗣️ **白話說明：** 就像開一家飲料店，不是第一天調好配方就永遠不變。要看客人反應（驗證）、改配方（重訓練）、換新品（重定義問題）。外送平台的演算法也一樣——每隔一段時間就要重新訓練，因為使用者行為一直在變。

**Quick check：**
- 「ML 流程第幾步才用到測試集？」→ 第 5 步（測試步驟）
- 「模型上線後效能下滑，應該回到哪一步？」→ 資料蒐集或模型訓練（重新訓練）

### 3.4 訓練集 / 驗證集 / 測試集

#### 先懂一句話：
> 全部資料分成三份，分別用來「學習」「調整」「最終評估」，三者不能重疊。

典型分割比例（僅為例示）：**7:1.5:1.5** 或 **8:1:1**

| 資料集 | 用途 | 可重複使用？ | 學生比喻 |
|---|---|---|---|
| 訓練集（Training Set） | 讓模型**學習**、調整內部**參數** | ✅ 可以 | 平時練習題 |
| 驗證集（Validation Set） | 訓練**過程中**調整**超參數**、比較模型 | ✅ 可以 | 模擬考 |
| 測試集（Test Set） | 訓練**完成後**做**最終評估** | ⚠️ 只能用一次 | 正式大考 |

🔥 **最重要的鐵律：三個資料集必須完全不重疊！若測試集和訓練集有重疊 = 作弊（資料洩漏）。**

🗣️ **白話說明：** 準備大學入學考試的版本——
- **訓練集** = 你平常刷的題庫（天天做、可以反覆練）
- **驗證集** = 模擬考（考完可以回去調整讀書方法）
- **測試集** = 學測正式考（考了就是考了，不能重來；也不能把考試結果拿來「再調整讀書方法後重考」）

**Quick check：**
- 「調整超參數應該看哪個資料集的表現？」→ 驗證集
- 「比例只能是 7:1.5:1.5 嗎？」→ 不是，比例依資料量調整

### 3.5 過度擬合 vs 擬合不足

#### 先懂一句話：
> 過度擬合（Overfitting）= 訓練高、測試低（背答案）；擬合不足（Underfitting）= 訓練低、測試也低（沒讀書）。

參考：`diagrams/03-bias-variance-tradeoff.md`

| | 過度擬合（Overfitting） | 擬合不足（Underfitting） | Good Fit |
|---|---|---|---|
| 訓練集表現 | **非常好**（近乎完美） | **差** | 好 |
| 測試集表現 | **差**（落差大） | **差**（和訓練集差不多差） | 好 |
| 本質 | 把雜訊和偶然細節也背下來 | 還沒學到真正的規律 | 學到真正的規律 |
| 常見原因 | 模型太複雜、訓練資料太少 | 模型太簡單、訓練不足 | — |
| 學生比喻 | 死背考古題，題目一變形就翻車 | 根本沒讀書，兩邊都爛 | 真正理解概念 |
| 解法方向（僅認名） | 加資料、降複雜度、提前停止、正則化 | 增複雜度、延長訓練、改善特徵 | — |

🗣️ **白話說明：**
- **過度擬合** = 有個學生把 108 課綱考古題的答案全部死背——只要考一模一樣的題就拿滿分，題目改一個數字就完全不會。
- **擬合不足** = 有個學生根本沒讀書，練習考 52 分，模擬考 51 分，兩邊都爛。

**Quick check：**
- 訓練集 99%、測試集 58% → **過度擬合**
- 訓練集 55%、測試集 53% → **擬合不足**
- 訓練集 93%、測試集 91% → **Good Fit**

### 3.6 模型評估指標

#### 先懂一句話：
> 單用準確率（Accuracy）評估模型可能誤導；Precision 和 Recall 從不同角度衡量分類效果，F1 是兩者的平衡。

⚠️ **初級邊界：知道每個指標的「用途和白話意義」即可；不需要記計算公式。**

參考：`diagrams/02-model-evaluation-metrics.md`

#### 混淆矩陣（Confusion Matrix，混淆矩陣）

一個 2×2 表格，視覺化分類模型的預測結果：

```
                    預測：正 (Positive)    預測：負 (Negative)
實際：正 (Positive)  TP（真正例）           FN（假負例）
實際：負 (Negative)  FP（假正例）           TN（真負例）
```

| 格子 | 全名 | 白話意思 |
|---|---|---|
| TP（True Positive，真正例） | 預測「是」、實際也「是」 | 正確抓到的正例 |
| TN（True Negative，真負例） | 預測「否」、實際也「否」 | 正確放過的負例 |
| FP（False Positive，假正例） | 預測「是」、實際是「否」 | 誤報（冤枉好人） |
| FN（False Negative，假負例） | 預測「否」、實際是「是」 | 漏報（放走壞人） |

#### 四大指標白話說明

| 指標 | 白話意義 | 關心什麼情境 |
|---|---|---|
| 準確率（Accuracy，準確率） | 全部預測中答對的比例 | 類別均衡時有意義 |
| 精確率（Precision，精確率） | 預測為「正」的那些人裡，真的「正」的比例 | 「誤報代價大」時優先（例：垃圾信誤判重要信） |
| 召回率（Recall，召回率） | 實際為「正」的所有人裡，被模型「找出來」的比例 | 「漏報代價大」時優先（例：癌症篩檢漏診） |
| F1 分數（F1 Score） | Precision 和 Recall 的調和平均，兩者兼顧 | 兩者都重要、想要平衡時 |

🗣️ **白話說明：**
- **Precision（精確率）= 警察抓人，抓了 10 人裡有幾個是真的壞人**（被抓的人裡，壞人比例）
- **Recall（召回率）= 所有壞人裡，警察抓到了幾個**（壞人被抓的比例）
- 想「少抓冤枉人」→ 提高 Precision；想「不放走任何壞人」→ 提高 Recall
- **F1 = 兩者都重要時的平衡分數**

**Quick check：**
- 「醫院做癌症篩檢，哪個指標更重要？」→ Recall（不能漏診任何一個真的癌症患者）
- 「垃圾郵件過濾，哪個指標更重要？」→ Precision（不能把重要郵件標成垃圾）

---

### 3.7 參數 vs 超參數

#### 先懂一句話：
> 參數（Parameter）是模型自己學出來的；超參數（Hyperparameter）是人類在訓練前設定的。

| | 參數（Parameter） | 超參數（Hyperparameter） |
|---|---|---|
| 誰設定 | **模型自己學** | **人類事先決定** |
| 何時決定 | 訓練**過程中** | 訓練**開始之前** |
| 可被訓練改變 | ✅ 會（訓練就是在調它） | ❌ 不會（除非人類手動改再重訓） |
| 學生比喻 | 學生腦中記下的知識 | 老師決定的讀書方法（每天讀幾小時） |
| 烤蛋糕比喻 | 烤好後的口感與組織 | 烤箱溫度、烤多久、加多少糖 |

🗣️ **白話說明：**
- **超參數** = 考前老師幫你規劃的讀書計畫（讀幾遍、幾點睡覺）——人類決定
- **參數** = 你讀完之後真正記在腦子裡的知識——你自己學到的

**口訣：人設的是超，自學的是參。**

**Quick check：** 「學習率（Learning Rate）是參數還是超參數？」→ 超參數（訓練前由人類設定）

---

### 3.8 批次學習 vs 線上學習

#### 先懂一句話：
> 批次學習（Batch Learning）一次訓練完才上線；線上學習（Online Learning）邊上線邊從新資料持續更新。

| | 批次學習（Batch Learning） | 線上學習（Online Learning，線上學習） |
|---|---|---|
| 學習方式 | 用全部訓練資料一次訓練完 | 每來一筆（或一小批）新資料就更新模型 |
| 更新頻率 | 定期重新訓練（通常週/月） | 近乎即時 |
| 適合情境 | 資料不常變、精準度優先 | 資料快速變化、需要即時跟上 |
| 例子 | 每季重訓的信用評分模型 | 串流平台即時推薦、詐欺偵測系統 |

🗣️ **白話說明：** Netflix 電影推薦如果是批次學習，就是「每個月更新一次你的推薦清單」。線上學習則是「你剛看完一部片，推薦清單立刻更新」。

---

### 3.9 特徵工程

#### 先懂一句話：
> 特徵工程（Feature Engineering，特徵工程）是把原始資料轉換成模型更容易學習的形式。

包含三個核心動作：

| 動作 | 說明 | 例子 |
|---|---|---|
| 特徵選擇（Feature Selection） | 從眾多特徵中挑出有用的 | 刪除與預測目標無關的欄位 |
| 特徵萃取（Feature Extraction） | 從原始資料中提取新的特徵 | 從照片中提取邊緣、顏色分布 |
| 特徵轉換（Feature Transformation） | 把特徵轉換成更好的格式 | 數值標準化、分類變數編碼 |

🗣️ **白話說明：** 你要訓練模型預測手搖飲銷量，原始資料有「日期」欄位。特徵工程就是把「2026-07-15」拆成「星期三」「暑假期間」「15 號（月中）」——這些加工後的特徵比原始日期更有用。

---

### 3.10 交叉驗證（Cross-Validation）

#### 先懂一句話：
> 交叉驗證（Cross-Validation，交叉驗證）是把訓練資料分成 K 份輪流當驗證集，讓評估結果更可靠。

`K 折交叉驗證（K-Fold Cross-Validation）`：把訓練資料分成 K 等份；每次用 K-1 份訓練、1 份驗證；重複 K 次，取平均成績。

用途：在資料量較少時，充分利用資料做更穩定的評估。

⚠️ 初級範圍：認得這個名詞 + 知道是為了「更可靠的評估」即可，不需要計算細節。

## Section 4 · 易混淆概念對照表

### 對照一：監督式 vs 非監督式

| 比較點 | 監督式學習 | 非監督式學習 |
|---|---|---|
| 標籤 | ✅ 需要 | ❌ 不需要 |
| 任務 | 分類、回歸 | 分群、降維 |
| 輸出 | 預測值（類別或數字） | 結構（群組或壓縮表示） |
| 考題關鍵詞 | 「有標籤」「預測」「分類正確率」 | 「無標籤」「分群」「探索性」 |

**考題判斷：** 看到「已知答案的資料來學習」→ 監督式；看到「沒有答案、自行找結構」→ 非監督式。

---

### 對照二：驗證集 vs 測試集

| 比較點 | 驗證集（Validation Set） | 測試集（Test Set） |
|---|---|---|
| 使用時機 | 訓練**過程中** | 訓練**完成後** |
| 用途 | 調整超參數、比較模型 | 最終評估泛化能力 |
| 可否重複使用 | ✅ 可以 | ⚠️ 原則上只能一次 |
| 可否用結果調模型 | ✅ 可以（這就是用途） | ❌ 不行（否則 = 資料洩漏） |
| 學生比喻 | 模擬考 | 學測正式考 |

**考題判斷：** 「訓練中反覆使用」→ 驗證集；「最後一次評估、只用一次」→ 測試集。

---

### 對照三：Precision vs Recall

| 比較點 | 精確率（Precision） | 召回率（Recall） |
|---|---|---|
| 白話 | 預測為正的人裡，有多少是真的正 | 所有真正的正例裡，模型找出了多少 |
| 關心的「壞事」 | FP（誤報、冤枉好人）多 | FN（漏報、放走壞人）多 |
| 優先使用情境 | 「誤報代價高」——垃圾信誤判重要信 | 「漏報代價高」——癌症篩檢漏診 |
| 兩者關係 | 通常是 Trade-off（提高一個會降低另一個） | ← 同左 |

**考題判斷：** 「不能漏診、寧可多送醫」→ Recall 優先；「不能誤判、寧可漏掉幾個」→ Precision 優先。

---

### 對照四：參數 vs 超參數

| 比較點 | 參數（Parameter） | 超參數（Hyperparameter） |
|---|---|---|
| 誰決定 | 模型自己在訓練中學 | 人類在訓練前設定 |
| 訓練時會變嗎 | ✅ 會 | ❌ 不會 |
| 例子 | 模型內部權重 | 訓練輪數、決策樹深度 |

**考題判斷：** 「訓練之前就要先填入的設定」→ 超參數；「訓練完成後才有值的」→ 參數。

---

### 對照五：Overfitting vs Underfitting

| 症狀 | 過度擬合（Overfitting） | 擬合不足（Underfitting） |
|---|---|---|
| 訓練集 | 高 | 低 |
| 測試集 | 低（落差大） | 低（和訓練差不多） |
| 模型特性 | 太複雜 | 太簡單 |
| 直覺 | 背答案 | 沒讀書 |

**考題判斷：** 訓練高、測試低（高低差大）→ Overfitting；兩邊都低 → Underfitting。

  非（無督）= 非監督式學習：無標籤，自己找規律
  強（獎勵）= 強化學習：靠獎懲，試錯學習

```
  練 → 訓練集（平時練習題，反覆用）
  模 → 驗證集（模擬考，訓練中調整）
  正 → 測試集（正式大考，只考一次）
```

聯想句：「**練**習後考**模**擬，準備好再上**正**式考場。」

### Overfitting vs Underfitting — 「高低差看過擬，兩邊低看欠擬」

```
  訓練高、測試低 → 過度擬合（高低差大 = 背答案）
  訓練低、測試低 → 擬合不足（兩邊都爛 = 沒讀書）
  訓練高、測試高 → Good Fit（泛化良好 = 真學會）
```

### 參數 vs 超參數 — 「人設超、自學參」

```
  人設的是超參數 → 人類事先設定
  自學的是參數  → 模型訓練中自己學到
```

### ML 流程 6 步 — 「蒐特訓驗測部」

```
  蒐 = 資料蒐集
  特 = 特徵工程
  訓 = 模型訓練
  驗 = 驗證調整
  測 = 測試評估
  部 = 部署監控
```

### Precision vs Recall — 「精確怕誤報，召回怕漏報」

```
  Precision（精確率）= 怕誤報（FP）多 → 警察抓人要精準
  Recall（召回率）  = 怕漏報（FN）多 → 癌症篩檢要全找
  F1               = 兩者都要平衡
```

## Section 6 · 考試陷阱

❌ **陷阱 1：驗證集 = 測試集（名字不同，用途相同）**
✅ 完全不同。驗證集在訓練**過程中**反覆用來調超參數；測試集在訓練**完成後**只用一次做最終評估。拿測試集結果回頭調模型 = 資料洩漏。

---

❌ **陷阱 2：過度擬合 = 模型沒學好**
✅ 剛好相反！過度擬合是模型學得「太好」——連訓練資料的雜訊都記下來了，所以訓練集表現**超好**，但測試集**反而差**。擬合不足才是「學不好（兩邊都差）」。

---

❌ **陷阱 3：非監督式學習 = 沒有監督、沒有目標，什麼都學不到**
✅ 非監督式學習同樣有明確的目標（找群組、降維），只是不需要人工標籤，而是從資料的內在結構中學習。

---

❌ **陷阱 4：準確率（Accuracy）是最好的評估指標**
✅ 資料類別不均衡時，Accuracy 會誤導。例：99% 的資料都是正常交易，模型全部預測「正常」也能得到 99% Accuracy，但詐欺一件都沒抓到。此時 Recall 更重要。

---

❌ **陷阱 5：監督式學習 = 有老師就是監督式**
✅ 「監督式」的「監督」指的是**標籤資料**，不是字面上有老師在旁邊。重點是訓練資料**有沒有正確答案標籤**。

---

❌ **陷阱 6：比例一定要 7:1.5:1.5**
✅ 7:1.5:1.5（或 70/15/15）只是常見例示，不是硬性規定。實際比例依資料量和問題調整。資料量極大時，驗證集比例可以很低，但絕對數量仍然夠用。

---

❌ **陷阱 7：Recall 高 = Precision 也高**
✅ Precision 和 Recall 通常是 Trade-off（此消彼長）。提高 Recall（漏報更少）往往會增加 FP（誤報），導致 Precision 下降。F1 是為了在兩者之間找平衡。

---

❌ **陷阱 8：強化學習一定要有大量標籤資料**
✅ 強化學習**不需要**預先標記的資料集；它透過和環境互動，從獎勵信號中學習。這和需要標籤的監督式學習完全不同。

---

❌ **陷阱 9：訓練集表現好 = 模型好**
✅ 訓練集高分只是基本盤——模型「看過」答案，拿高分是應該的。真正重要的是**測試集**（未見過資料）的表現，也就是**泛化能力（Generalization）**。

---

❌ **陷阱 10：特徵工程是模型訓練的一部分，由模型自動完成**
✅ 傳統 ML 的特徵工程由**人類設計**，在訓練**之前**完成。它是資料準備的一環，不是模型自己學的（深度學習可以自動學習特徵，但那是中級的主題）。

---

## Section 7 · 情境題快速判斷

### 7.1 決策流程圖

```
遇到情境題，先問：

「有沒有標籤資料？」
         │
    ┌────┴─────┐
    │有         │沒有
    ▼           ▼
監督式學習    「是在試錯找最佳策略嗎？」
（分類/回歸）      │
             ┌───┴───┐
             │是      │否
             ▼        ▼
         強化學習   非監督式
         （試錯）  （分群/降維）

若已知模型在訓練集/測試集表現：
  訓練高、測試低 → Overfitting（過度擬合）
  訓練低、測試低 → Underfitting（擬合不足）
  兩者都高       → Good Fit

若問評估指標選哪個：
  「漏了代價大」（癌症、詐欺）→ Recall 優先
  「誤報代價大」（垃圾信過濾）→ Precision 優先
  「都要兼顧」              → F1 Score
```

### 7.2 🔑 關鍵詞 → 答案快查表

| 情境關鍵詞 | 答案 |
|---|---|
| 有標籤資料、預測類別 | 監督式學習（分類） |
| 有標籤資料、預測數值 | 監督式學習（回歸） |
| 無標籤、自動分組 | 非監督式學習（分群） |
| 代理人、環境、獎勵、試錯 | 強化學習 |
| 訓練集 99%、測試集 60% | 過度擬合（Overfitting） |
| 訓練集 58%、測試集 56% | 擬合不足（Underfitting） |
| 訓練集 91%、測試集 89% | Good Fit（泛化良好） |
| 調整超參數用哪個資料集 | 驗證集 |
| 最終評估用哪個資料集 | 測試集（只用一次） |
| 「不能漏診」癌症篩檢 | Recall 優先 |
| 「不能誤判」垃圾郵件 | Precision 優先 |
| 訓練前人類設定的值 | 超參數（Hyperparameter） |
| 模型訓練中自動調整的值 | 參數（Parameter） |
| 邊上線邊即時更新模型 | 線上學習（Online Learning） |
| 一次訓練完再上線 | 批次學習（Batch Learning） |
| 把資料分 K 份輪流驗證 | K 折交叉驗證（K-Fold Cross-Validation） |
| 預測為正但實際為負 | FP（假正例 False Positive） |
| 實際為正但模型沒抓到 | FN（假負例 False Negative） |
| ML 流程是直線還是循環 | 循環（iterative） |
| 垃圾郵件過濾 ML 或傳統 | 機器學習（規則會演化、寫不完） |

### 7.3 Mini 情境練習

**情境 A：**
一家醫院訓練 AI 模型來篩檢 X 光片中的肺癌，訓練集準確率 97%，測試集準確率 61%，而且很多真正的癌症患者被模型預測為正常。請問：(1) 模型遇到什麼問題？(2) 哪個評估指標在此情境中最關鍵？

**答案：** (1) 過度擬合（訓練高、測試低，高低差大）(2) Recall（召回率）——癌症不能漏診，FN 的代價遠大於 FP。

---

**情境 B：**
電商平台想分析一百萬名用戶的購買行為，找出不同消費族群，但沒有任何預設的「族群標籤」。這適合哪種學習範式？

**答案：** 非監督式學習（分群 Clustering）——無標籤資料，目標是探索自然群組。

---

**情境 C：**
某 ML 工程師在訓練完模型後，用測試集看了結果，覺得不滿意，於是根據測試集的結果回頭調整超參數，再次評估，如此重複了三次。這樣做有什麼問題？

**答案：** 資料洩漏（Data Leakage）。反覆用測試集結果調整模型，等於讓模型間接「學到」測試集的資訊，導致評估結果虛胖，上線後真實效能可能遠低於測試集分數。應使用**驗證集**來調整超參數，測試集只能在最終評估時使用一次。

---

**情境 D：**
AlphaGo 下圍棋，透過和自己的「鏡像」不斷對弈，每贏一局就得到正向回饋，輸了就得到負向回饋，逐漸提升棋力。這是哪種學習範式？

**答案：** 強化學習（Reinforcement Learning）——代理人（AlphaGo）、環境（棋盤）、獎勵（勝負）、試錯學習。

---

**情境 E：**
一個垃圾郵件過濾器，目前 Precision = 0.95，Recall = 0.60。這意味著什麼？應該優化哪個方向？

**答案：** Precision 0.95 表示被標記為垃圾的郵件中有 95% 確實是垃圾（誤報少）；但 Recall 0.60 表示只抓到了 60% 的垃圾郵件，有 40% 漏到收件匣。如果用戶更在意「垃圾郵件都被過濾掉」，應優化 Recall；如果用戶更在意「重要郵件不會被誤判」，現在的 Precision 已經很高。

---
