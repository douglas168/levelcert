# 圖示 02：模型評估指標與混淆矩陣

> 對應 Study Guide Section 3.6

## 混淆矩陣（Confusion Matrix）2×2 視覺化

```
                   ┌──────────────────────────────────────────────────────┐
                   │                  模型預測結果                          │
                   │                                                        │
                   │           預測：正 (Predicted Positive)               │
                   │           預測：負 (Predicted Negative)               │
    ┌──────────────┼────────────────────────┬───────────────────────────┤
    │              │  預測：正              │  預測：負                  │
    │  實際：正     │  TP（真正例）           │  FN（假負例）              │
    │  (Actual +)  │  True Positive         │  False Negative            │
    │              │  ✅ 預測正、實際正       │  ❌ 預測負、實際正（漏報）   │
    ├──────────────┼────────────────────────┼───────────────────────────┤
    │  實際：負     │  FP（假正例）           │  TN（真負例）              │
    │  (Actual -)  │  False Positive        │  True Negative             │
    │              │  ❌ 預測正、實際負（誤報）│  ✅ 預測負、實際負          │
    └──────────────┴────────────────────────┴───────────────────────────┘
```

## 四個格子的白話說明

| 格子 | 全名 | 白話 | 垃圾信過濾例子 |
|---|---|---|---|
| TP（True Positive，真正例） | 預測「是」、實際「是」 | 正確抓到的壞人 | 垃圾信被正確標記為垃圾 |
| TN（True Negative，真負例） | 預測「否」、實際「否」 | 正確放過的好人 | 正常信被正確放行 |
| FP（False Positive，假正例） | 預測「是」、實際「否」 | 誤報（冤枉好人） | 正常信被誤判為垃圾 |
| FN（False Negative，假負例） | 預測「否」、實際「是」 | 漏報（放走壞人） | 垃圾信漏進收件匣 |

## 四大評估指標

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  準確率 Accuracy = 全部預測中答對的比例                                        │
│  白話：「所有情況下，猜對了幾成？」                                             │
│  ⚠️ 類別不均衡時可能誤導（見下方陷阱）                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│  精確率 Precision = 預測為「正」的人裡，真的「正」的比例                         │
│  白話：「我標記的壞人裡，真的有幾成是壞人？」                                    │
│  關心：FP（誤報）多不多 → 誤報代價高時優先                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│  召回率 Recall = 實際為「正」的所有人裡，被找出來的比例                          │
│  白話：「所有真正的壞人裡，我抓到了幾成？」                                      │
│  關心：FN（漏報）多不多 → 漏報代價高時優先                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│  F1 分數 F1 Score = Precision 和 Recall 的調和平均                            │
│  白話：「Precision 和 Recall 的平衡分數」                                      │
│  適用：兩者都重要、想要兼顧時                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Precision vs Recall 情境選擇

```
         漏報代價 >> 誤報代價？
         （放走壞人比冤枉好人更糟？）
                    │
          ┌─────────┴─────────┐
          │ 是                 │ 否（誤報代價更高）
          ▼                   ▼
      Recall 優先         Precision 優先
      （不能漏掉任何）      （不能誤傷任何）
      癌症篩檢、          垃圾郵件過濾、
      詐欺偵測            新聞內容審核
                │
                ▼
        兩者都重要？→ F1 Score
```

## ⚠️ 常見考試陷阱

| 陷阱 | 正確理解 |
|---|---|
| Accuracy 高 = 模型好 | ❌ 類別不均衡時，全猜多數類別就能高 Accuracy，但 Recall = 0 |
| Precision 高 = Recall 也高 | ❌ 兩者通常是 Trade-off（此消彼長） |
| Recall 高 = 誤報少 | ❌ 正好相反！Recall 高通常意味著 FP 增加（誤報更多） |
