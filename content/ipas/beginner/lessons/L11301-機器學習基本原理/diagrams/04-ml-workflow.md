# 圖示 04：機器學習工作流程（ML Workflow）

> 對應 Study Guide Section 3.3

## 六步驟循環流程圖

```
  ┌────────────────────────────────────────────────────────────────────────┐
  │                    ML 工作流程（循環，非直線）                            │
  └────────────────────────────────────────────────────────────────────────┘

  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
  │  1. 資料蒐集  │──▶│  2. 特徵工程  │──▶│  3. 模型訓練  │
  │  Data        │    │  Feature     │    │  Model       │
  │  Collection  │    │  Engineering │    │  Training    │
  └──────────────┘    └──────────────┘    └──────────────┘
         ▲                                        │
         │                                        ▼
  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
  │  6. 部署監控  │◀──│  5. 測試評估  │◀──│  4. 驗證調整  │
  │  Deploy &    │    │  Test Set    │    │  Validation  │
  │  Monitor     │    │  Evaluation  │    │  & Tuning    │
  └──────────────┘    └──────────────┘    └──────────────┘
         │
         │ 概念漂移（Concept Drift）
         │ 效能退化時
         └──────────────────────────────────────▶ 回到步驟 1 或 3
```

## 六步驟說明表

| 步驟 | 名稱 | 主要工作 | 關鍵原則 |
|---|---|---|---|
| 1 | 資料蒐集（Data Collection） | 收集、清理、整理原始資料 | 品質 > 數量；garbage in → garbage out |
| 2 | 特徵工程（Feature Engineering） | 選擇/萃取/轉換有用特徵 | 好特徵 = 讓模型更容易學到規律 |
| 3 | 模型訓練（Model Training） | 用訓練集讓模型學習、調整參數 | 訓練集不能和驗證集/測試集重疊 |
| 4 | 驗證調整（Validation & Tuning） | 用驗證集調整超參數、比較模型 | 反覆使用驗證集是合法的 |
| 5 | 測試評估（Test Evaluation） | 用測試集做最終評估 | 測試集只能用一次（保持聖潔性） |
| 6 | 部署監控（Deploy & Monitor） | 上線、追蹤效能 | 監控概念漂移，必要時重訓 |

## 🔥 循環的觸發點

```
  步驟 5（測試結果很差）
  → 回到步驟 3（換模型）或步驟 2（改特徵）或步驟 1（加資料）

  步驟 6（上線後效能下滑 / 概念漂移）
  → 回到步驟 1（重新收集新資料）→ 重新訓練整個流程

  注意：不能用測試集結果回頭調超參數！
  如果要調超參數 → 回到步驟 4（用驗證集調整）
```

## 流程記憶口訣

```
  蒐（資料蒐集）
  特（特徵工程）
  訓（模型訓練）
  驗（驗證調整）
  測（測試評估）
  部（部署監控）

  → 蒐特訓驗測部，走完還要回頭循環
```

## ⚠️ 高頻考試陷阱

| 陷阱說法 | 正確觀念 |
|---|---|
| ML 工作流程是一次性的，從頭跑到尾就完成了 | ❌ 流程是循環的，評估不佳或效能退化都要回頭 |
| 訓練完成後就不需要再管模型了 | ❌ 部署後要持續監控，處理概念漂移 |
| 用測試集結果回去調超參數沒問題 | ❌ 那是資料洩漏，調超參數應用驗證集 |
| ML 工作流程的第一步是訓練模型 | ❌ 第一步是資料蒐集，訓練是第三步 |
